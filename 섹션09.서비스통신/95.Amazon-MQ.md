## 1. Amazon MQ란?

Amazon MQ는 **Apache ActiveMQ** 및 **RabbitMQ**를 위한 **관리형 메시지 브로커 서비스**입니다.

### 핵심 배경

* **SQS/SNS:** AWS 전용 프로토콜과 API를 사용하는 '클라우드 네이티브' 서비스입니다.
* **기존 환경(On-Premise):** 대중적인 개방형 프로토콜(MQTT, AMQP 등)을 사용합니다.
* **도입 이유:** 온프레미스에서 클라우드로 마이그레이션할 때, 기존 애플리케이션의 코드를 수정(Re-platforming/Refactoring)하지 않고 그대로 사용하기 위해 도입합니다.

---

## 2. 주요 프로토콜 및 호환성

Amazon MQ는 다양한 표준 메시징 프로토콜을 지원하여 기존 시스템과의 연결성이 뛰어납니다.

* **지원 프로토콜:** AMQP, MQTT, STOMP, OpenWire, WSS
* **교체 가능한 서비스:**
  * IBM MQ
  * TIBCO EMS
  * RabbitMQ
  * Apache ActiveMQ



---

## 3. Amazon MQ의 특징

### 1) 관리형 서비스

* ActiveMQ와 RabbitMQ 브로커의 설치, 운영, 유지보수를 AWS가 대신 관리합니다.

### 2) 기능의 통합

* **SQS의 대기열(Queue)** 기능과 **SNS의 토픽(Topic)** 기능을 하나의 브로커 안에서 모두 제공합니다.

### 3) 가용성 및 확장성 (SQS/SNS와의 차이점)

* **확장성:** SQS/SNS처럼 무한대에 가깝게 확장되지는 않습니다. (서버 기반 서비스의 한계)
* **가용성:** 서버 장애에 대비하여 **Multi-AZ(다중 가용 영역)** 배치를 통해 장애 조치(Failover) 기능을 지원합니다.

---

## 4. 요약: SQS vs Amazon MQ (시험 포인트)

| 구분 | Amazon SQS / SNS | Amazon MQ |
| --- | --- | --- |
| **성격** | 클라우드 네이티브 (AWS 전용) | 기존 기술의 관리형 버전 (Open Source 기반) |
| **주요 목적** | 신규 클라우드 앱 구축 | **기존 앱의 빠른 마이그레이션** |
| **프로토콜** | AWS 전용 API | MQTT, AMQP, STOMP 등 (개방형) |
| **확장성** | 거의 무제한 (Serverless) | 특정 인스턴스 성능에 의존 |

---

### 💡 시험 대비 팁

시험에서 **"기존 애플리케이션을 수정하지 않고(Minimal code changes) 마이그레이션해야 한다"**는 조건과 함께 **"MQTT, AMQP"** 같은 단어가 등장한다면 정답은 높은 확률로 **Amazon MQ**입니다.
