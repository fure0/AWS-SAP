# 🖥️ Amazon WorkSpaces 핵심 요약

Amazon WorkSpaces는 클라우드 기반의 **관리형 보안 가상 데스크톱 서비스(VDI)**입니다. 온프레미스 VDI 구축에 필요한 복잡한 인프라 관리 부담을 줄여주며, 높은 보안성과 확장성을 제공합니다.

---

## 1. 기본 특징 및 요금제

* **OS 지원:** Windows 및 Linux 가상 데스크톱 제공.
* **연동:** Microsoft Active Directory(AD)와 통합하여 도메인 가입 및 권한 관리 가능.
* **요금 옵션:**
* **시간제(Hourly):** 사용한 시간만큼만 비용 지불 (간헐적 사용자 유리).
* **월 구독(Monthly):** 고정 비용으로 무제한 사용 (상시 사용자 유리).


* **보안:** 망 분리 환경 제공, 데이터 암호화 지원.

---

## 2. 주요 관리 도구: WAM (WorkSpaces Application Manager)

애플리케이션의 배포와 관리를 효율화하는 도구입니다. **OS 업데이트와는 별개**로 작동한다는 점이 중요합니다.

* **가상화 컨테이너:** 애플리케이션을 가상 컨테이너화하여 배포.
* **확장성:** 모든 WorkSpaces 작업 공간에 신속하게 앱 배포 및 최신 상태 유지.
* **차이점:** Windows 업데이트는 '장치 전체'를 대상으로 하지만, **WAM은 오직 '애플리케이션'만을 대상**으로 합니다.

---

## 3. 실행 모드 및 유지 관리 (Maintenance)

Windows 업데이트 및 시스템 유지 보수는 설정된 '유지 관리 기간'에 수행됩니다.

| 실행 모드 | 유지 관리 특징 |
| --- | --- |
| **AlwaysOn** | 항상 켜져 있으며, 매주 일요일 00:00 ~ 04:00 사이에 자동 업데이트 수행. |
| **AutoStop** | 미사용 시 중단됨. 사용 빈도가 낮을 경우 업데이트 주기가 월 1회로 자동 조정됨. |

> **참고:** 유지 관리 기간 중에는 WorkSpace가 일시적으로 **오프라인** 상태가 될 수 있습니다.

---

## 4. 교차 리전 리디렉션 (Cross-Region Redirection)

재해 복구(DR) 및 고가용성을 위해 여러 리전에 WorkSpaces를 배치하는 아키텍처입니다.

* **구성 요소:**
* **기본 리전:** AWS Managed Microsoft AD 사용.
* **복구 리전:** **AD Connector** 사용 (현재 멀티 리전 Microsoft AD가 지원되지 않기 때문).
* **Route 53:** 연결 별칭(Connection Alias)과 연동된 TXT 레코드(`desktop.example.com`)를 생성하여 트래픽 유도.
* **상태 확인(Health Check):** 기본 리전 장애 발생 시 Route 53이 복구 리전의 코드를 반환하여 사용자 접속을 유도.


* **데이터 동기화:** WorkSpaces는 리전별로 독립적입니다. 사용자 데이터를 리전 간 공유하려면 **Amazon WorkDocs** 같은 별도 서비스를 활용해야 합니다.

---

## 5. 보안 및 액세스 제어

* **IP 액세스 제어 그룹 (IP Access Control Groups):** * WorkSpaces 전용 보안 그룹(Security Group) 역할을 합니다.
* 신뢰할 수 있는 공인 IP 주소 또는 CIDR 범위만 접속 허용.
* **주의:** VPN이나 NAT을 거쳐 접속하는 경우, 해당 **VPN/NAT Gateway의 공인 IP**를 반드시 허용 리스트에 추가해야 합니다.
