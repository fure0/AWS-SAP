## AWS AppStream 2.0 개요

**AppStream 2.0**은 개별 **데스크톱 애플리케이션**을 사용자에게 스트리밍으로 제공하는 완전관리형 서비스입니다.

* **핵심 개념:** 별도의 인프라 구축 없이, 웹 브라우저를 통해 데스크톱 앱을 다른 기기로 전송합니다.
* **작동 방식:** 애플리케이션은 AWS 서버에서 실행되고, 화면 정보만 사용자의 웹 브라우저(Chrome 등)로 스트리밍됩니다.
* **장점:**
  * **브라우저 기반:** 웹 브라우저가 있는 어떤 기기(Windows, MacOS, 태블릿 등)에서도 실행 가능합니다.
  * **호환성:** Windows 전용 앱을 Mac 환경에서 브라우저를 통해 그대로 사용할 수 있습니다.



---

## 주요 특징: 유연한 자원 할당

AppStream 2.0의 가장 큰 장점 중 하나는 **애플리케이션별 최적화**가 가능하다는 점입니다.

* **맞춤형 리소스:** 실행하려는 앱의 사양에 맞춰 CPU, 메모리, GPU 자원을 조정할 수 있습니다.
* **예시:**
  * **일반 문서 도구:** 낮은 사양의 컨테이너 할당.
  * **그래픽 도구(Photoshop, Blender):** 높은 CPU 및 GPU 자원을 가진 컨테이너 할당.

## AppStream 2.0 vs Amazon WorkSpaces 비교

시험에서 가장 자주 묻는 두 서비스의 차이점을 표로 정리했습니다.

| 구분 | Amazon WorkSpaces | AWS AppStream 2.0 |
| --- | --- | --- |
| **서비스 형태** | **VDI (가상 데스크톱 인프라)** | **애플리케이션 스트리밍** |
| **제공 범위** | 운영체제(OS)를 포함한 전체 데스크톱 환경 | 특정 애플리케이션(앱) 자체만 제공 |
| **사용 방식** | 가상 PC에 로그인하여 여러 앱 사용 | 웹 브라우저를 통해 특정 앱만 실행 |
| **리소스 관리** | 인스턴스 단위의 사양 고정 경향 | 앱의 요구 사양에 맞춰 컨테이너 자원 조정 가능 |
| **주요 목적** | 직원의 업무용 PC 대체 | 특정 앱(설계, 그래픽, 교육용 등)의 원격 배포 |

---

> **💡 핵심 요약**
> * **WorkSpaces**는 "나의 가상 컴퓨터"를 통째로 빌리는 것과 같습니다.
> * **AppStream 2.0**은 컴퓨터 전체가 아닌, "필요한 프로그램"만 브라우저로 띄워서 쓰는 방식입니다.
