# 🚀 Amazon CodeGuru 핵심 정리

Amazon CodeGuru는 **머신 러닝(ML)** 기술을 활용하여 코드 품질을 높이고 애플리케이션의 성능을 최적화하는 자동화 서비스입니다. 크게 두 가지 핵심 기능인 **Reviewer**와 **Profiler**로 나뉩니다.

---

## 1. CodeGuru Reviewer (코드 리뷰 자동화)

개발자가 소스 코드를 저장소에 푸시하면, AI가 사람이 놓치기 쉬운 결함을 찾아내는 **정적 코드 분석** 도구입니다.

* **주요 기능:**
* **자동 코드 리뷰:** CodeCommit, GitHub, BitBucket과 연동되어 코드를 자동으로 분석합니다.
* **결함 감지:** 버그, 메모리 누수(Memory Leak), 보안 취약점 등을 식별합니다.
* **최적화 제안:** 단순한 오류 지적을 넘어, 구체적이고 현실적인 수정 방안을 제시합니다.


* **작동 원리:** * Amazon.com의 내부 코드 저장소와 수천 개의 오픈 소스 프로젝트 데이터를 학습한 ML 모델 및 **자동화된 추론(Automated Reasoning)** 기술을 사용합니다.
* **지원 언어:** Java, Python (현재 기준)

---

## 2. CodeGuru Profiler (실행 성능 최적화)

애플리케이션이 실행 중인 상태(런타임)에서 자원 사용량을 분석하여 효율성을 극대화하는 **성능 진단** 도구입니다.

* **주요 기능:**
* **비용 효율성:** CPU 사용량이 비정상적으로 높은 '가장 비싼 코드(Most expensive lines of code)'를 식별하여 컴퓨팅 비용을 절감합니다.
* **리소스 분석:** 힙(Heap) 요약을 제공하여 메모리를 과도하게 점유하는 객체를 찾아냅니다.
* **이상 징후 감지:** 로깅 작업 시 CPU 과부하 등 비정상적인 동작을 실시간으로 감지합니다.


* **특징:**
* **낮은 오버헤드:** 애플리케이션 성능에 거의 영향을 주지 않으면서 모니터링이 가능합니다.
* **환경 유연성:** AWS 클라우드뿐만 아니라 **온프레미스(On-premise)** 환경에서 실행되는 앱도 지원합니다.



---

## 🔍 Reviewer vs Profiler 비교 요약

| 구분 | CodeGuru Reviewer | CodeGuru Profiler |
| --- | --- | --- |
| **분석 시점** | 코드 작성 및 빌드 단계 (Static) | 애플리케이션 실행 단계 (Runtime) |
| **주요 목적** | 버그 수정, 보안 강화, 모범 사례 준수 | CPU/메모리 효율화, 비용 절감 |
| **핵심 이점** | 사람이 놓치는 논리적 오류 감지 | 운영 환경의 병목 현상 제거 |

---

> **💡 시험 및 실무 팁**
> * **Reviewer**는 "코드가 저장소에 올라갈 때(Push/Pull Request)" 동작하는 **품질 관리자**라고 생각하세요.
> * **Profiler**는 "앱이 돌아가고 있을 때(Test/Production)" 동작하는 **성능 감별사**라고 이해하면 쉽습니다.
