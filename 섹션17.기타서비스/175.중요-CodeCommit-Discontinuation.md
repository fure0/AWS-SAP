## 📢 AWS CodeCommit 서비스 업데이트 및 가이드

최근 AWS의 정책 변화에 따라 CodeCommit 사용 방식에 큰 변화가 생겼습니다. 시험을 준비하시는 분들은 아래 내용을 반드시 숙지하시기 바랍니다.

### 1. 주요 공지 사항 (Service Deprecation)

* **서비스 중단 결정:** 2024년 7월 25일부터 AWS는 CodeCommit 서비스의 신규 기능 개발 및 신규 고객 유치를 중단했습니다.
* **신규 고객 사용 불가:** 해당 날짜 이후로 AWS 계정을 생성한 경우, CodeCommit 서비스를 활성화하거나 사용할 수 없습니다.
* **권장 사항:** AWS는 현재 **GitHub, GitLab** 또는 기타 타사 Git 솔루션으로의 마이그레이션을 강력히 권장하고 있습니다.

---

### 2. AWS SAP 시험에서의 관점

서비스는 사실상 은퇴 수순을 밟고 있지만, **자격증 시험(SAP 등)**에서는 여전히 등장할 수 있습니다.

* **시험 출제 가능성:** CodeCommit은 여전히 시험 문제에 포함될 수 있으므로 개념은 학습해야 합니다.
* **대체제로서의 이해:** CodeCommit이 문제에 나온다면, 단순히 **"AWS 환경 내에서 사용하는 관리형 GitHub/Git 저장소"**라고 이해하시면 충분합니다.
* **유연한 사고:** 강의나 문제에서 CodeCommit 대신 **GitHub 통합**이 언급되더라도 당황하지 마세요. 두 서비스는 소스 코드를 관리하는 동일한 역할을 수행합니다.

---

### 3. 핵심 요약 테이블

| 구분 | 내용 |
| --- | --- |
| **서비스 역할** | AWS 관리형 소스 제어 서비스 (Git 기반) |
| **현재 상태** | 2024년 7월 25일 이후 신규 고객 지원 중단 |
| **대체 솔루션** | GitHub, GitLab, Bitbucket 등 타사 Git 서비스 |
| **학습 전략** | 기능을 깊게 파기보다는 **CI/CD 파이프라인의 시작점**이라는 개념 위주로 파악 |

---

> **💡 학습 팁:**
> 앞으로 강의 중에 CodeCommit이 언급되더라도 "아, 저건 그냥 Git 저장소구나"라고 가볍게 생각하고 넘어가셔도 무방합니다. 실제 실무에서는 GitHub이나 GitLab 마이그레이션 시나리오가 더 중요해질 것입니다.
