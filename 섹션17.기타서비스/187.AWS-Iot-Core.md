## ## AWS IoT Core 핵심 요약

IoT(Internet of Things)는 데이터를 수집하고 전송하는 '사물'들이 네트워크로 연결된 것을 의미합니다. AWS IoT Core는 이러한 수많은 장치들을 클라우드에 **안전하고 확장 가능하게** 연결해 주는 **서버리스** 서비스입니다.

### ### 주요 특징

* **서버리스(Serverless):** 인프라 관리 없이 수십억 개의 장치와 수조 개의 메시지 처리가 가능합니다.
* **게이트웨이 역할:** 장치와 클라우드 서비스(Lambda, S3, SageMaker 등) 사이의 가교 역할을 합니다.
* **보안 및 확장성:** 대규모 장치 연결에 최적화된 보안 통신을 지원합니다.

---

## ## 데이터 통신 방식: Pub/Sub 모델

IoT Core는 **발행(Publish) 및 구독(Subscribe)** 개념을 사용하여 장치나 애플리케이션 간의 통신을 처리합니다.

* **프로토콜:** 주로 **MQTT** 프로토콜을 사용하여 메시지를 주고받습니다.
* **IoT 주제(Topic):** SNS Topic과 유사한 방식으로 데이터를 분류하고 수신하는 채널 역할을 합니다.

---

## ## IoT 규칙(Rules) 및 작업(Action)

메시지가 IoT 주제로 전달되면, 미리 설정한 **IoT 규칙**에 따라 데이터를 처리합니다.

### ### 규칙 내 '작업(Action)'이란?

수신된 메시지를 어떤 AWS 서비스로 보낼지 정의하는 단계입니다. 이를 통해 데이터를 분석하거나 저장소로 전달할 수 있습니다.

**연동 가능한 주요 서비스:**

* **분석/스트리밍:** Kinesis, MSK
* **데이터베이스:** DynamoDB
* **메시징:** SQS, SNS
* **컴퓨팅/저장:** Lambda, S3

---

## ## 시험 대비 필수 연동 패턴: Kinesis Data Firehose

시험에서는 특히 **IoT Core와 Kinesis Data Firehose의 연동 흐름**을 이해하는 것이 중요합니다.

### ### 데이터 흐름도

1. **MQTT 메시지 수신:** IoT Core가 장치로부터 데이터를 받습니다.
2. **Firehose 전달:** IoT 규칙을 통해 메시지가 거의 실시간으로 **Kinesis Data Firehose**로 전달됩니다.
3. **데이터 가공 (옵션):** Firehose 내부에서 **AWS Lambda**를 호출하여 데이터를 변환하거나 가공합니다.
4. **최종 저장:** 가공된 데이터를 다음 목적지에 보관합니다.
* **Amazon S3** (데이터 레이크)
* **Amazon Redshift** (데이터 웨어하우스)
* **Amazon OpenSearch** (검색 및 분석)



---

> **핵심 포인트:** IoT Core는 단순히 연결만 해주는 것이 아니라, **'규칙(Rule)'**을 통해 다양한 AWS 서비스로 데이터를 실시간 라우팅하는 중심축 역할을 한다는 점을 기억하세요!
