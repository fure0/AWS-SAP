## 💡 AWS Device Farm 핵심 정리

**AWS Device Farm**은 모바일 및 웹 애플리케이션의 품질을 보장하기 위해 실제 장치에서 테스트를 수행하는 **애플리케이션 테스트 서비스**입니다.

### 1. 주요 특징

* **실제 장치 기반 테스트:** 에뮬레이터가 아닌, 클라우드에 있는 **실제(Real) 모바일 장치 및 브라우저**에서 앱을 실행합니다.
* **완전 자동화:** 다양한 테스트 프레임워크를 사용하여 여러 장치에서 동시에 테스트를 자동화할 수 있습니다.
* **광범위한 환경 지원:** Android, iOS 기기는 물론 다양한 웹 브라우저 환경을 지원합니다.

### 2. 주요 기능 및 장점

* **품질 향상:** 사용자가 실제로 사용하는 다양한 하드웨어/OS 버전에서의 동작을 검증하여 앱의 신뢰성을 높입니다.
* **자동 분석 및 문서화:**
  * 테스트 실패 시 **동영상 및 로그가 자동으로 생성**됩니다.
  * 장애 발생 원인을 시각적으로 확인하고 문서화하기 용이합니다.

* **원격 액세스(Remote Access):** 브라우저를 통해 실시간으로 장치에 원격 로그인하여 직접 조작하며 **디버깅**을 수행할 수 있습니다.

---

### 3. 테스트 결과 확인 (예시)

Device Farm은 대시보드를 통해 장치별 성공/실패 여부를 명확히 구분해 줍니다.

| 장치 모델 | 테스트 결과 | 비고 |
| --- | --- | --- |
| **HTC Phone** | ✅ Pass | 정상 작동 확인 |
| **Nexus 6** | ❌ Fail | 문제 발생 (로그 확인 필요) |
| **LG G Flex 2** | ❌ Fail | 문제 발생 (로그 확인 필요) |
| **Motorola** | ❌ Fail | 특정 환경에서의 장애 발생 |

> **💡 시험 포인트:**
> "다양한 실제 모바일 기기에서의 호환성 테스트"나 "자동화된 UI 테스트" 요구사항이 나오면 **AWS Device Farm**을 정답으로 고려하세요.
