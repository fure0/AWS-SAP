## 🔐 AWS Certificate Manager (ACM) 요약 정리

ACM은 웹 사이트와 애플리케이션을 보호하기 위한 **SSL/TLS 인증서**를 손쉽게 발급, 관리 및 배포할 수 있게 해주는 서비스입니다.

---

### 1. 인증서 확보 방법

ACM을 통해 인증서를 관리하는 방법은 크게 두 가지입니다.

* **ACM 프로비저닝 (권장):** ACM을 통해 직접 인증서를 발급받는 방식입니다.
  * **비용:** 무료입니다.
  * **갱신:** **자동으로 갱신**되므로 관리가 매우 편리합니다.


* **외부 인증서 가져오기 (Import):** 외부 기관에서 직접 구매한 인증서를 CLI 등을 통해 업로드하는 방식입니다.
  * **갱신:** 자동 갱신이 지원되지 않으며, 사용자가 **수동으로 관리**해야 합니다.



---

### 2. ACM과 통합되는 서비스

ACM 인증서는 SSL/TLS 종료(Termination)가 필요한 다양한 AWS 서비스와 통합됩니다.

* **Elastic Load Balancing (ALB, NLB, CLB):** Elastic Beanstalk에 의해 생성된 로드 밸런서 포함.
* **Amazon CloudFront:** 글로벌 콘텐츠 전송 네트워크.
* **Amazon API Gateway:** API 엔드포인트 보안.

---

### 3. 작동 방식: SSL 종료 (SSL Termination)

1. **사용자 ↔ ALB:** 사용자는 HTTPS를 통해 ALB와 통신합니다.
2. **인증서 관리:** ACM이 ALB에 SSL 인증서를 프로비저닝하고 유지 관리합니다.
3. **SSL 종료:** ALB 단계에서 암호화된 연결이 종료(복호화)됩니다.
4. **ALB ↔ EC2:** ALB는 백엔드 EC2 인스턴스로 프라이빗 HTTP 요청을 전달합니다.
   * **장점:** EC2 인스턴스가 복잡한 복호화 과정을 처리할 필요가 없어 **CPU 부하가 감소**합니다.



---

### 4. 공인 인증서 vs 프라이빗 인증서

| 구분 | 공인 인증서 (Public) | 프라이빗 인증서 (Private) |
| --- | --- | --- |
| **용도** | 인터넷을 통한 외부 서비스용 | 내부 애플리케이션 및 인트라넷용 |
| **신뢰 기관** | 공인 인증 기관(CA)에서 발급 | 자체 프라이빗 CA 생성 |
| **특이 사항** | 퍼블릭 DNS 확인 절차 필요 | 애플리케이션이 프라이빗 CA를 신뢰하도록 별도 설정 필수 |

---

### 5. 💡 시험 및 실무 중요 포인트 (Must Know)

#### **1) 리전 서비스 (Regional Service)**

* ACM은 **리전별로 독립적**인 서비스입니다.
* 여러 리전에 ALB가 배포되어 있다면, **각 리전마다 인증서를 개별적으로 발급**받아야 합니다.
* 리전 간에 인증서를 복사하거나 이동할 수 없습니다.

#### **2) CloudFront의 경우**

* CloudFront는 글로벌 서비스이지만, ACM 인증서를 사용하려면 반드시 **us-east-1(버지니아 북부) 리전**에서 인증서를 발급/요청해야 합니다.

#### **3) 자동 갱신 여부**

* ACM에서 직접 프로비저닝한 인증서만 자동 갱신됩니다. 외부에서 가져온(Imported) 인증서는 만료 전 사용자가 직접 개입해야 한다는 점을 명심하세요.
