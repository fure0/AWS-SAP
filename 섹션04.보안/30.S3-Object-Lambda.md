## S3 Object Lambda (객체 람다) 개요

**S3 Object Lambda**는 S3 버킷에 저장된 객체를 애플리케이션이 검색할 때, **데이터를 실시간으로 수정하거나 처리**하여 전달하는 기능입니다.

기존에는 동일한 데이터의 여러 버전(예: 원본 버전, 마스킹 버전, 압축 버전 등)이 필요할 경우 각각의 버킷을 새로 만들어야 했지만, Object Lambda를 사용하면 **단일 S3 버킷**만으로도 요청자의 요구에 맞는 다양한 형태의 데이터를 즉석에서 생성할 수 있습니다.

---

### 작동 원리 및 구성 요소

S3 Object Lambda가 구동되기 위해서는 다음과 같은 흐름이 필요합니다.

1. **S3 버킷 (S3 Bucket):** 실제 오리지널 데이터가 저장된 장소입니다.
2. **S3 액세스 포인트 (S3 Access Point):** 버킷 앞에 위치하여 람다가 데이터에 접근할 수 있는 통로 역할을 합니다.
3. **람다 함수 (Lambda Function):** 요청된 객체를 가져와서 코드를 실행(수정, 변환, 삭제 등)하는 본체입니다.
4. **S3 객체 람다 액세스 포인트 (S3 Object Lambda Access Point):** 최종 애플리케이션이 접근하는 엔드포인트입니다.

---

### 주요 활용 사례 (Use Cases)

강의에서 언급된 시나리오별 활용 방법은 다음과 같습니다.

| 활용 시나리오 | 설명 |
| --- | --- |
| **데이터 마스킹 (PII 삭제)** | 분석 애플리케이션이 접근할 때, 개인식별정보(PII)를 실시간으로 삭제하여 보안을 유지함. |
| **데이터 보강 (Enrichment)** | 마케팅 애플리케이션 요청 시, 외부 데이터베이스(예: 고객 충성도 DB)에서 정보를 가져와 객체에 추가함. |
| **포맷 변환** | XML 형식의 데이터를 즉석에서 JSON으로 변환하여 반환함. |
| **이미지 처리** | 요청자에 맞게 실시간으로 이미지 크기를 조정하거나 워터마크를 삽입함. |

---

### 워크플로우 예시: 분석 및 마케팅 팀

1. **전자상거래 앱:** S3 버킷에 직접 액세스하여 **오리지널 객체**를 그대로 읽고 씁니다.
2. **분석기 앱:** "객체 람다 액세스 포인트"를 통해 접근합니다. 연결된 **람다 함수**가 PII 정보를 삭제한 뒤 가공된 데이터를 전달합니다.
3. **마케팅 앱:** 별도의 "객체 람다 액세스 포인트"를 사용합니다. **람다 함수**가 외부 DB에서 데이터를 가져와 객체 내용을 보강한 뒤 전달합니다.

> **핵심 요약:** > "여러 개의 버킷을 복사할 필요 없이, 하나의 원본 버킷 위에서 **S3 액세스 포인트 + 람다 함수**를 조합하여 요청에 따라 데이터를 실시간 변환하는 기술"입니다.
