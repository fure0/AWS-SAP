## 1. SSM Parameter Store 개요

SSM Parameter Store는 애플리케이션의 설정 데이터와 비밀번호, 연결 문자열 등의 민감한 정보를 안전하게 저장하고 관리하는 **서버리스 저장소**입니다.

* **주요 특징:**
* **서버리스 및 확장성:** 별도의 인프라 관리 없이 높은 가용성과 확장성을 제공합니다.
* **버전 관리:** 파라미터 업데이트 시 버전을 추적하여 이전 설정을 확인할 수 있습니다.
* **보안:** **IAM**을 통해 접근 권한을 제어하며, **KMS**와 연동하여 데이터를 암호화합니다.
* **통합 환경:** CloudFormation 스택의 입력 파라미터로 활용 가능하며, EventBridge와 연동하여 상태 변화 알림을 받을 수 있습니다.



---

## 2. 작동 방식 및 계층 구조

Parameter Store는 데이터를 단순한 리스트가 아닌 **계층적 경로(Path)**로 관리할 수 있어 권한 관리가 매우 용이합니다.

### **데이터 암호화 프로세스**

1. **평문 저장:** 일반적인 설정값 저장.
2. **암호화 저장:** KMS 키를 사용하여 데이터를 암호화합니다.
3. **애플리케이션 접근:** EC2나 Lambda 등의 IAM 역할(Role)에 Parameter Store 접근 권한 및 해당 KMS 키에 대한 사용 권한이 모두 있어야 복호화하여 사용할 수 있습니다.

### **계층 구조 관리 (Hierarchical Structure)**

경로 형식을 사용하여 부서별, 애플리케이션별, 환경별로 파라미터를 구분합니다.

* **예시:**
* `/my-app/dev/db-url`
* `/my-app/prod/db-url`


* **장점:** IAM 정책에서 특정 경로(예: `/my-app/dev/*`)에 대해서만 권한을 부여함으로써 보안을 단순화하고 강화할 수 있습니다.

---

## 3. 유용한 기능 및 트릭

* **Secrets Manager 참조:** Parameter Store 인터페이스를 통해 Secrets Manager에 저장된 비밀값에 접근할 수 있습니다.
* **퍼블릭 파라미터 (Public Parameters):** AWS가 직접 관리하고 발행하는 파라미터입니다.
* **활용 예:** 특정 리전의 최신 **Amazon Linux 2 AMI ID**를 API 호출 한 번으로 조회하여 자동화 스크립트에 활용할 수 있습니다.



---

## 4. 파라미터 등급 비교: Standard vs Advanced

사용 목적과 데이터 크기에 따라 두 가지 등급을 선택할 수 있습니다.

| 구분 | Standard (스탠다드) | Advanced (어드밴스드) |
| --- | --- | --- |
| **최대 크기** | 4 KB | 8 KB |
| **비용** | 무료 | 월 $0.05 / 파라미터당 |
| **파라미터 정책** | 지원하지 않음 | **지원 (TTL 등)** |
| **저장 용량** | 최대 10,000개 | 최대 100,000개 |

---

## 5. 파라미터 정책 (Advanced 전용)

Advanced 등급에서만 사용 가능한 기능으로, 파라미터의 생명 주기를 관리합니다.

* **만료 정책 (Expiration/TTL):** 특정 날짜나 시간에 파라미터가 삭제되도록 설정합니다. 보안을 위해 비밀번호 업데이트를 강제할 때 유용합니다.
* **알림 설정 (EventBridge 통합):**
* **만료 예고:** 삭제 15일 전 알림을 받아 파라미터를 갱신할 시간을 확보합니다.
* **변경 없음 알림 (NoChangeNotification):** 일정 기간(예: 20일) 동안 업데이트되지 않은 파라미터가 있다면 알림을 보냅니다.
