## 💡 AWS 관리형 로그 서비스 정리

AWS의 주요 서비스들은 자체적으로 상세한 로그를 생성하며, 이를 분석을 위해 **S3, CloudWatch, Kinesis** 등으로 내보낼 수 있습니다.

### 1. 네트워크 및 로드 밸런싱 관련 로그

* **Load Balancer Access Logs (ALB, NLB, CLB)**
  * **내용:** 로드 밸런서로 들어오는 모든 요청의 세부 정보(클라이언트 IP, 대기 시간, 경로 등)를 기록합니다.
  * **대상:** **S3**로 내보내어 분석할 수 있습니다.


* **VPC Flow Logs**
  * **내용:** VPC 내 네트워크 인터페이스(ENI)를 통과하는 IP 트래픽(In/Out)에 대한 정보를 캡처합니다.
  * **대상:** **S3, CloudWatch Logs, Kinesis Data Firehose**로 전송 가능합니다.


* **Route 53 Access Logs**
  * **내용:** Route 53이 수신하는 모든 DNS 쿼리에 대한 정보를 기록합니다.
  * **대상:** **CloudWatch Logs**로 직접 전송 및 스트리밍됩니다.



### 2. 계정 활동 및 보안 로그

* **AWS CloudTrail Logs**
  * **내용:** AWS 계정 내에서 발생하는 모든 API 호출 내역을 기록합니다. (누가, 언제, 어떤 리소스를 조작했는지 확인)
  * **대상:** **S3** 및 **CloudWatch Logs**로 내보낼 수 있습니다.


* **AWS Config Information**
  * **내용:** 리소스의 구성 변경 이력을 기록하며, 규정 준수 여부를 확인하는 데 사용됩니다.
  * **대상:** 구성 백업 및 분석을 위해 **S3**로 전송됩니다.



### 3. 콘텐츠 전송 및 스토리지 로그

* **S3 Access Logs**
  * **내용:** 특정 S3 버킷에 대한 모든 액세스 요청(객체 업로드, 다운로드 등) 정보를 제공합니다.
  * **대상:** 분석을 위해 **다른 S3 버킷**으로 전송합니다.


* **CloudFront Access Logs**
  * **내용:** 전 세계 Edge Location에서 발생하는 모든 사용자 요청에 대한 상세 정보를 기록합니다.
  * **대상:** **S3**에 저장됩니다.

## 📊 로그별 대상 저장소 요약표

| 로그 유형 | 대상 서비스 (Target) |
| --- | --- |
| **LB Access Logs** | S3 |
| **CloudTrail** | S3, CloudWatch Logs |
| **VPC Flow Logs** | S3, CloudWatch Logs, Kinesis Data Firehose |
| **Route 53** | CloudWatch Logs |
| **S3 Access Logs** | S3 (다른 버킷) |
| **CloudFront** | S3 |
| **AWS Config** | S3 |

---

> **Tip:**
> 시험에서는 특정 로그를 분석하기 위해 어떤 서비스를 연결해야 하는지(예: S3에 저장 후 Athena로 쿼리, 또는 CloudWatch를 통한 실시간 모니터링) 자주 묻습니다. 이 연결 고리를 잘 기억해 두세요!
