## 1. AWS Application Discovery Service

마이그레이션 계획 수립 단계에서 온프레미스 데이터센터의 정보를 수집하는 서비스입니다.

* **주요 기능:** 서버 활용도 데이터 수집, 서버 간 의존성 매핑(Migration 전 필수 작업).
* **수집 방식:**
1. **에이전트리스(Agentless):** `OVA 패키지` 형태로 VMware 호스트에 배포. OS에 상관없이 설정 및 성능 이력(CPU, 메모리 등) 조사.
2. **에이전트 기반(Agent-based):** 서버(Linux, Windows 등)에 직접 설치. 실행 중인 프로세스 및 **상세 네트워크 연결 정보**를 파악하여 전체 서비스 맵을 그려냄.


* **데이터 활용:** 수집된 데이터는 CSV로 내보내거나 **AWS Migration Hub**에서 확인 가능하며, **Amazon Athena**를 통해 쿼리 분석이 가능합니다.

---

## 2. Migration Hub Data Exploration

수집된 대규모 데이터를 분석하고 가시화하는 도구입니다.

* **분석 도구:** **Amazon Athena**를 사용하여 S3에 저장된 서버 데이터를 쿼리(예: 실행 중인 프로세스 종류 파악).
* **추가 소스:** 기존에 보유한 CMDB(설정 관리 데이터베이스) 데이터를 업로드하여 통합 분석 가능.
* **가시화:** **Amazon QuickSight**와 Athena를 연동하여 대시보드 형태로 시각화.

---

## 3. AWS Application Migration Service (MGN)

실제 서버를 AWS로 옮기는 **리호스트(Re-host, Lift-and-Shift)** 솔루션입니다.

* **히스토리:** 기존의 `SMS(Server Migration Service)`와 `CloudEndure Migration`이 통합된 서비스입니다. (시험에서 구 명칭이 나오면 MGN으로 이해)
* **동작 원리:**
1. 온프레미스 서버에 **복제 에이전트** 설치.
2. **스테이징 환경**으로 데이터 연속 복제 (EBS 볼륨에 저장).
3. 준비 완료 시 운영 환경으로 **컷오버(Cutover)** 하여 정식 EC2 인스턴스로 전환.


* **장점:** 최소한의 다운타임, 다양한 OS/DB 지원, 비용 절감.

---

## 4. AWS Elastic Disaster Recovery (DRS)

재해 복구(DR)를 위한 서비스로, MGN과 기술적 뿌리(`CloudEndure DR`)가 같습니다.

* **목적:** 단순 이관이 아닌 **재해 시 빠른 복구**가 주 목적.
* **대상:** Oracle, MySQL, SQL Server, SAP 등 중요 기업용 애플리케이션 및 랜섬웨어 대비 보호.
* **특징:**
* 초 단위의 데이터 복제(RPO 최소화).
* 재해 발생 시 몇 분 안에 AWS로 **페일오버(Failover)** 수행.
* 온프레미스 복구 시 **페일백(Failback)** 지원.

## 5. 기타 관련 서비스 요약

| 서비스명 | 주요 용도 | 비고 |
| --- | --- | --- |
| **Amazon Linux 2 AMI** | VM 이미지 다운로드 | VMware, Hyper-V 등 온프레미스 환경에서 실행 가능 |
| **AWS Migration Hub** | 프로젝트 트래킹 | 이관 상태를 한곳에서 관리하고 추적 |
| **AWS DMS** | 데이터베이스 마이그레이션 | DB 전용(Oracle, MySQL, DynamoDB 등), 양방향 복제 가능 |

시험에서는 **"의존성 파악"**이 키워드라면 **Discovery Service**를, **"서버 전체 리호스트(Lift-and-Shift)"**가 목적이라면 **MGN**을 선택하세요. 데이터베이스만 옮긴다면 **DMS**가 정답일 확률이 높습니다.
