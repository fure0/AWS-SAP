## ## AWS Migration Evaluator 개요

**AWS Migration Evaluator**는 조직이 AWS 마이그레이션을 결정할 때, 추측이 아닌 **데이터 기반의 계획**과 **비즈니스 사례(Business Case)**를 구축할 수 있도록 지원하는 서비스입니다.

### ### 주요 목적

* **워크로드 파악:** 현재 운영 중인 인프라의 정확한 상태 확인
* **비용 인사이트:** AWS 전환 시 예상되는 비용 및 가성비 분석
* **의사결정 지원:** 경영진이나 의사결정권자를 위한 명확한 근거 자료 생성

---

## ## 마이그레이션 평가 프로세스

### 1. 데이터 수집 (Discovery)

가장 먼저 현재 온프레미스 환경의 '풋프린트(Footprint)'와 서버 간의 '의존성'을 파악해야 합니다.

* **Agentless Collector 설치:** 대상 서버에 일일이 에이전트를 깔지 않고도 광범위한 탐색 작업을 수행합니다.
* **스냅샷 촬영:** 인프라 현황과 서버 의존성을 데이터 형태로 캡처합니다.
* **데이터 임포트:** 기존에 관리하던 데이터가 있다면 제공되는 **도구와 템플릿**을 사용하여 직접 데이터를 업로드할 수도 있습니다.

### 2. 현재 상태 분석 및 대상 상태 정의

* 수집된 데이터를 바탕으로 현재 온프레미스 소비 리소스를 분석합니다.
* 이를 AWS의 어떤 서비스(대상 상태, Target State)로 옮길지 정의하고 구체적인 마이그레이션 계획을 수립합니다.

### 3. 비즈니스 사례 구축 및 결과 도출

* **비용 효율성:** 마이그레이션 시의 가성비를 분석하여 최적의 비용 모델을 제시합니다.
* **전문 가이드:** 필요한 경우 AWS 전문가가 분석된 데이터를 바탕으로 전문적인 비즈니스 사례 가이드를 제공합니다.

## ## 핵심 요약 테이블

| 단계 | 주요 활동 | 활용 도구 |
| --- | --- | --- |
| **수집** | 광범위한 인프라 탐색 및 서버 의존성 파악 | Agentless Collector, 데이터 임포트 템플릿 |
| **분석** | 온프레미스 현황 분석 및 AWS 최적화 모델링 | Migration Evaluator 분석 엔진 |
| **결과** | 비용 인사이트 도출 및 비즈니스 사례 수립 | AWS 전문 가이드 및 리포트 |

---

> **💡 핵심 포인트**
> 마이그레이션 평가기는 단순히 "옮길 수 있다"를 넘어, **"얼마나 효율적으로, 어떤 이득을 보며 옮길 수 있는가"**에 대한 답을 주는 서비스입니다.
