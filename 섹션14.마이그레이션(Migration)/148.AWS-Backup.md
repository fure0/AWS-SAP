# ☁️ AWS Backup 서비스 완전 정복

**AWS Backup**은 AWS 리소스 전체에 대해 데이터 보호를 중앙에서 관리하고 자동화할 수 있는 **완전 관리형 서비스**입니다.

---

## 1. 핵심 개념 및 특징

* **중앙 집중식 관리:** 여러 서비스에 흩어진 백업 작업을 한 곳에서 통제합니다. (수동 스크립트 불필요)
* **완전 자동화:** 백업 스케줄링 및 보존 정책을 자동화하여 운영 부담을 줄입니다.
* **교차 기능 지원:**
  * **교차 리전 백업(Cross-Region Backup):** 재해 복구(DR)를 위해 다른 리전으로 백업본 복사.
  * **교차 계정 백업(Cross-Account Backup):** AWS Organizations를 활용하여 다른 계정으로 백업본 복사.

* **유연한 백업 방식:**
  * **온디맨드 백업:** 필요할 때 즉시 실행.
  * **스케줄 백업:** 정의된 주기(Cron 표현식 가능)에 따라 자동 실행.
  * **태그 기반 백업:** 특정 태그(예: `Env: Production`)가 지정된 리소스만 선별하여 백업.



---

## 2. 지원되는 주요 서비스

AWS Backup은 지속적으로 지원 범위를 넓혀가고 있으며, 현재 핵심적인 대상은 다음과 같습니다.

| 분류 | 지원 서비스 |
| --- | --- |
| **컴퓨팅/스토리지** | EC2, EBS, Amazon S3, EFS |
| **데이터베이스** | RDS(모든 엔진), Aurora, DynamoDB, DocumentDB, Neptune |
| **파일 시스템/기타** | Amazon FSx(Lustre, Windows), Storage Gateway(Volume Gateway) |

> **참고:** Aurora와 같이 서비스 자체에서 지원하는 경우 **PITR(Point-in-Time Recovery)** 기능도 AWS Backup을 통해 활용 가능합니다.

---

## 3. 백업 플랜(Backup Plan) 설정 요소

백업 플랜은 "어떻게, 언제, 얼마나 오래" 백업할지를 정의하는 정책입니다.

* **백업 주기:** 12시간마다, 매주, 매달 또는 **Cron 표현식**을 통한 커스텀 설정.
* **백업 창(Backup Window):** 백업이 시작되고 수행될 시간대 설정.
* **라이프사이클 관리:**
  * **Cold Storage 이동:** 비용 절감을 위해 일정 기간 후 콜드 스토리지로 전환 (며칠/몇 주/몇 년 단위 설정).
  * **보존 기간(Retention):** 백업본을 얼마나 유지할지 결정 (영구 보관 가능).



---

## 4. 보안 및 데이터 보호 (Vault Lock)

강력한 보안이 필요한 환경에서 가장 중요한 기능입니다.

### **Backup Vault Lock 이란?**

* **WORM(Write Once, Read Many) 모델 적용:** 한 번 기록되면 수정이나 삭제가 불가능합니다.
* **삭제 방지:** 실수로 인한 삭제나 악의적인 공격(랜섬웨어 등)으로부터 백업 데이터를 보호합니다.
* **루트 권한 보호:** Vault Lock이 활성화되면 **루트(Root) 사용자조차 백업을 삭제하거나 보존 기간을 단축할 수 없습니다.**

---

## 5. 백업 프로세스 요약

1. **백업 플랜 생성:** 주기, 보존 기간, 리전 복사 여부 정의.
2. **리소스 할당:** 태그 또는 리소스 ID를 통해 백업 대상을 지정.
3. **데이터 저장:** 백업 데이터는 AWS Backup용 내부 S3 버킷(사용자 접근 불가 영역)에 안전하게 저장됩니다.
