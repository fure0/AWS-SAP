## ☁️ 클라우드 마이그레이션 전략: 7R

마이그레이션은 단순히 서버를 옮기는 것 이상의 전략적 선택이 필요합니다. AWS에서 제시하는 7가지 전략은 다음과 같습니다.

### 1. Retire (폐기)

* **정의:** 더 이상 필요하지 않은 애플리케이션이나 기능을 **종료**하는 전략입니다.
* **장점:**
  * 공격 표면적(Attack Surface)을 줄여 보안 강화.
  * 불필요한 리소스 삭제로 약 10~20%의 비용 절감.
  * 유지 관리가 필요한 핵심 리소스에 집중 가능.



### 2. Retain (유지)

* **정의:** 현재 상태를 유지하며 **온프레미스에 그대로 두는** 전략입니다.
* **이유:**
  * 데이터 규정 준수(Compliance)나 엄격한 보안 요구사항.
  * 해결되지 않은 복잡한 종속성 문제.
  * 마이그레이션 실익(비즈니스 가치)이 낮거나 비용이 너무 많이 드는 경우.



### 3. Relocate (이전)

* **정의:** 앱 아키텍처를 변경하지 않고 인스턴스를 **새로운 환경(VPC, 계정, 리전 등)으로 이동**하는 것입니다.
* **예시:** 온프레미스의 VMware 환경을 **VMware Cloud on AWS**로 그대로 이동하는 경우. (설정 변경 최소화)

### 4. Rehost (리호스트 - "Lift and Shift")

* **정의:** 최적화 과정 없이 애플리케이션을 **있는 그대로** 클라우드로 옮기는 가장 단순한 방법입니다.
* **특징:**
  * 물리/가상 서버를 EC2로 바로 마이그레이션.
  * **AWS Application Migration Service(MGN)**를 주로 사용.
  * 빠른 이전이 가능하며, 클라우드 리소스 활용만으로도 약 30% 비용 절감 효과 기대.



### 5. Replatform (리플랫폼 - "Lift and Reshape")

* **정의:** 핵심 아키텍처는 유지하되, 일부 요소를 **클라우드 최적화 서비스(Managed Service)**로 교체하는 전략입니다.
* **예시:**
  * 온프레미스 DB 서버를 직접 설치하는 대신 **Amazon RDS**로 변경.
  * 애플리케이션을 **Elastic Beanstalk** 환경으로 이동.

* **장점:** 자동 백업, 고가용성 등 관리형 서비스의 이점을 즉시 누릴 수 있음.

### 6. Repurchase (재구매 - "Drop and Shop")

* **정의:** 기존 제품을 버리고 **SaaS(Software as a Service)** 형태의 다른 제품으로 전환하는 전략입니다.
* **예시:**
  * 자체 구축 CRM → **Salesforce**
  * 자체 인사 시스템 → **Workday**
  * 자체 CMS → **Drupal** 기반 서비스

* **특징:** 초기 비용이나 라이선스 비용이 들 수 있으나 배포가 매우 빠름.

### 7. Refactor / Re-architect (리팩터 / 재설계)

* **정의:** 클라우드 네이티브 기능을 100% 활용하기 위해 **애플리케이션 아키텍처를 완전히 다시 설계**하는 전략입니다.
* **특징:**
  * 모놀리식 구조를 **마이크로서비스(MSA)**로 분해.
  * Serverless(Lambda), S3 등 현대적 기술 적용.
  * **가장 많은 노력과 시간**이 소요되지만, 확장성·성능·민첩성 면에서 **가장 큰 성과**를 얻음.



---

### 💡 요약 비교표

| 전략 | 복잡도 | 클라우드 최적화 수준 | 주요 특징 |
| --- | --- | --- | --- |
| **Retire** | 낮음 | N/A | 필요 없는 서비스 제거 (비용 절감) |
| **Retain** | 낮음 | N/A | 온프레미스 유지 (규제/복잡성 이슈) |
| **Relocate** | 낮음 | 낮음 | VMware 등을 AWS 환경으로 이동 |
| **Rehost** | 낮음 | 낮음 | 가장 빠른 이동 (Lift & Shift) |
| **Replatform** | 중간 | 중간 | 관리형 서비스(RDS 등) 도입 |
| **Repurchase** | 중간 | 높음 | SaaS 제품으로 교체 |
| **Refactor** | 높음 | 높음 | 클라우드 네이티브 재설계 (최대 효과) |

---

이 7가지 전략 중 시험 문제의 상황(예: "최소한의 변경으로 빠르게 이전하라" vs "최대 효율과 확장성을 확보하라")에 따라 적절한 R을 선택하는 것이 핵심입니다.
