## 1. AWS FIS란 무엇인가?

**AWS FIS**는 AWS 워크로드에 의도적으로 **오류를 주입(Fault Injection)**하여 시스템의 복원력을 테스트하는 완전 관리형 서비스입니다.

* **기반 기술:** **카오스 엔지니어링(Chaos Engineering)**에 기반을 둡니다.
* **핵심 원리:** 시스템에 인위적인 스트레스(장애 이벤트)를 가해 전체 애플리케이션 스택이 어떻게 대응하는지 확인합니다.
* **주요 예시:**
* CPU 또는 메모리 사용량의 급격한 과부하 유도
* 데이터베이스 인스턴스 장애 발생
* 네트워크 지연 시간 증가 등



### 💡 왜 FIS를 사용하나요?

1. **애플리케이션 견고성 확인:** 실제 장애 상황에서도 서비스가 유지되는지 검증합니다.
2. **숨겨진 버그 발견:** 일반적인 테스트로는 찾기 힘든 간헐적 오류를 식별합니다.
3. **성능 병목 현상 파악:** 시스템의 약한 고리를 미리 찾아내어 개선할 수 있습니다.

---

## 2. 지원하는 주요 서비스

FIS는 AWS의 다양한 네이티브 서비스와 통합되어 장애를 시뮬레이션합니다.

| 서비스 | 주요 장애 작업 (Action) |
| --- | --- |
| **Amazon EC2** | 인스턴스 강제 종료, 리부팅 등 |
| **Amazon ECS** | 실행 중인 ECS 태스크(Task) 중단 |
| **Amazon EKS** | Kubernetes 노드나 태스크 중단 |
| **Amazon RDS** | DB 인스턴스 장애 유도, 장애 조치(Failover) 테스트 |

---

## 3. FIS 작동 프로세스

실험은 아래와 같은 단계로 진행됩니다.

1. **실험 템플릿(Template) 생성:**
* 사전 정의된 템플릿을 사용하여 어떤 오류를 발생시킬지 설정합니다.


2. **대상 리소스(Target) 선택:**
* 오류를 심을 특정 EC2 인스턴스, ECS 클러스터, RDS DB 등을 지정합니다.


3. **실험 가동 및 스트레스 주입:**
* 실험이 시작되면 선택한 리소스에 실제 지장이 생기도록 명령을 내립니다.


4. **모니터링 및 관찰:**
* **CloudWatch:** 로그 및 지표 확인
* **EventBridge:** 이벤트 발생 여부 체크
* **X-Ray:** 서비스 간 호출 흐름 및 병목 구간 추적


5. **결과 확인 및 개선:**
* 복원력에 문제가 없는지 확인하고, 발견된 병목 현상을 해결하여 애플리케이션을 개선합니다.



---

## 4. 요약 및 결론

* **FIS는 고급 디버깅 기술:** 단순한 모니터링을 넘어 능동적으로 장애를 일으켜 시스템을 강화하는 도구입니다.
* **네이티브 통합:** 별도의 도구 설치 없이 AWS 환경에서 바로 사용할 수 있다는 것이 큰 장점입니다.

> **핵심 포인트:** FIS는 시스템을 **"일부러 망가뜨려 봄으로써"** 더 단단한 시스템을 만들기 위한 도구입니다.
