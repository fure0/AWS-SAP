# ❄️ AWS Snowball 서비스 완벽 정리

**AWS Snowball**은 엣지(Edge) 환경에서 데이터를 수집·처리하거나, AWS 안팎으로 대용량 데이터를 안전하게 마이그레이션할 수 있는 **휴대용 물리 디바이스**입니다.

---

## 1. Snowball Edge 디바이스 종류

사용 목적에 따라 두 가지 사양으로 나뉩니다. 가장 큰 차이점은 **저장 공간**과 **컴퓨팅 성능**의 비중입니다.

| 구분 | Snowball Edge 스토리지 최적화 | Snowball Edge 컴퓨팅 최적화 |
| --- | --- | --- |
| **핵심 용도** | 대규모 데이터 전송 및 보관 | 강력한 연산이 필요한 엣지 컴퓨팅 |
| **저장 용량** | **최대 210TB** | 약 28TB |
| **주요 특징** | 페타바이트급 마이그레이션에 적합 | 데이터 처리 및 분석 능력 강화 |

---

## 2. 데이터 마이그레이션 사용 사례

네트워크를 통해 직접 업로드하는 방식(S3 직접 업로드)보다 물리적 장치를 이용하는 것이 유리한 상황에서 사용합니다.

### 🚩 네트워크 전송의 한계 (예시)

* **1Gbps** 연결 속도로 **100TB**를 전송할 경우: 약 **12일** 소요.
* **문제점:** 대역폭 제한, 높은 네트워크 비용, 다른 서비스와 대역폭 공유로 인한 불안정성.

### ✅ Snowball 활용 프로세스

1. **신청:** AWS에서 물리적인 Snowball 기기를 주문하여 인프라에 설치.
2. **로드:** 로컬 데이터를 Snowball 장치에 직접 로드.
3. **운송:** 데이터가 담긴 장치를 AWS로 다시 배송.
4. **반입:** AWS 측에서 장치의 데이터를 추출하여 **Amazon S3 버킷**으로 업로드.

> **Tip:** 데이터 전송에 일주일 이상 걸릴 것으로 예상된다면 Snowball을 사용하는 것이 훨씬 효율적입니다.

---

## 3. 엣지 컴퓨팅(Edge Computing) 사용 사례

인터넷 연결이 제한적이거나 성능이 부족한 열악한 환경에서 현장 데이터를 즉시 처리해야 할 때 사용합니다.

### 📍 주요 환경 예시

* 도로 위의 **트럭**, 바다 위의 **선박**, 오지의 **채굴장** 등.

### 🛠️ 주요 기능 및 활용

* **컴퓨팅 실행:** 장치 내에서 직접 **Amazon EC2 인스턴스**나 **AWS Lambda 함수**를 실행 가능.
* **데이터 전처리:** 데이터가 생성되는 즉시 현장에서 처리.
* **머신러닝(ML):** 엣지 단에서 실시간 머신러닝 추론 수행.
* **미디어 트랜스코딩:** 비디오 파일 등을 현장에서 즉시 변환.

---

## 💡 요약 및 결론

AWS Snowball은 단순히 '큰 외장 하드'가 아니라, **강력한 보안**과 **컴퓨팅 성능**을 갖춘 이동식 데이터 센터입니다.

* **데이터 마이그레이션:** 대역폭 한계를 극복하고 페타바이트급 데이터를 전송할 때.
* **엣지 컴퓨팅:** 인터넷이 안 되는 곳에서 데이터를 수집하고 즉시 처리할 때.
