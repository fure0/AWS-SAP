## ## 1. AWS DataSync 개요

DataSync는 온프레미스, 타사 클라우드, 또는 AWS 서비스 간에 대규모 데이터를 빠르고 안전하게 이동 및 **동기화**하는 서비스입니다.

* **주요 용도:** 대규모 데이터 마이그레이션, 데이터 보호 및 복제, 하이브리드 클라우드 워크로드 처리.
* **작동 방식:** 실시간 지속 복제가 아닌 **예약된 태스크(Scheduled Tasks)** 기반 동기화 (시간/일/주 단위).
* **성능:** 단일 태스크당 최대 **10Gbps**의 속도를 지원하며, 필요에 따라 **대역폭 제한(Bandwidth Limiting)** 설정이 가능합니다.

---

## ## 2. 핵심 기능 및 특징

### 📋 파일 보존 및 보안

* **메타데이터 보존:** 파일 권한(NFS POSIX, SMB Permissions)과 메타데이터를 원본 그대로 보존합니다. (시험에서 매우 강조되는 포인트!)
* **암호화:** 데이터 전송 시 암호화되어 안전하게 이동합니다.

### 🌐 연결 프로토콜 및 대상

* **지원 프로토콜:** NFS, SMB, HDFS 등.
* **대상 스토리지:**
* **Amazon S3:** 모든 스토리지 클래스 지원 (Glacier 포함).
* **Amazon EFS:** 네트워크 파일 시스템.
* **Amazon FSx:** FSx for Windows File Server, Lustre 등 모든 종류 지원.

## ## 3. 데이터 동기화 아키텍처

### 🏢 Case 1: 온프레미스 ↔ AWS (에이전트 필요)

온프레미스 환경에서 데이터를 주고받을 때는 반드시 **DataSync 에이전트**를 설치해야 합니다.

1. **에이전트 설치:** 온프레미스 또는 타사 클라우드 서버에 설치.
2. **연결:** 에이전트가 NFS/SMB 서버에 연결하고 AWS DataSync 서비스와 암호화 통신 수행.
3. **양방향성:** 온프레미스 → AWS뿐만 아니라, AWS → 온프레미스로의 역방향 동기화도 지원합니다.

### ☁️ Case 2: AWS 서비스 ↔ AWS 서비스 (에이전트 불필요)

S3, EFS, FSx 간에 데이터를 이동할 때는 별도의 에이전트 설치 없이 DataSync 서비스를 통해 직접 복제 및 메타데이터 보존이 가능합니다.

### 📦 Case 3: 네트워크 대역폭이 부족한 경우 (Snowcone 활용)

인터넷 연결이 불안정하거나 대역폭이 부족할 때 사용합니다.

* **AWS Snowcone** 기기에는 DataSync 에이전트가 기본적으로 설치되어 있습니다.
* 현장에서 데이터를 Snowcone에 담아 AWS로 물리적으로 보낸 뒤 동기화하는 방식을 취합니다.

## ## 4. 시험 대비 핵심 요약

| 구분 | 주요 내용 |
| --- | --- |
| **복제 주기** | 예약 기반 (Incremental Sync), 실시간(Continuous) 아님 |
| **에이전트** | 온프레미스/타 클라우드 연결 시 필수, AWS 서비스 간 이동 시 불필요 |
| **강점** | **메타데이터 및 권한 보존**, 최대 10Gbps 고속 전송 |
| **특수 케이스** | 네트워크 용량 부족 시 **Snowcone + DataSync** 조합 사용 |
