# 🚀 AWS Compute Optimizer 핵심 정리

AWS Compute Optimizer는 **비용 절감**과 **성능 개선**을 동시에 달성하기 위한 서비스입니다. 머신러닝을 통해 사용자의 워크로드를 분석하고 최적의 리소스를 제안합니다.

## 1. 주요 목적 및 작동 원리

* **목적:** 워크로드에 가장 적합한(Optimal) 리소스를 권장하여 비용 최적화 및 성능 향상.
* **작동 방식:**
  * **머신러닝 분석:** 리소스 설정 및 사용 기록을 학습합니다.
  * **지표 트래킹:** **Amazon CloudWatch** 지표를 기반으로 사용 패턴을 파악합니다.
  * **결과 도출:** 분석을 통해 리소스가 **'프로비저닝 과다(Over-provisioned)'**인지 **'프로비저닝 부족(Under-provisioned)'**인지 판별합니다.



## 2. 지원 리소스 및 혜택

* **지원 대상:**
  * EC2 인스턴스
  * EC2 Auto Scaling 그룹
  * EBS(Elastic Block Store) 볼륨
  * AWS Lambda 함수

* **기대 효과:** 별도의 복잡한 작업 없이 **최대 25%의 비용 절감**이 가능합니다.
* **데이터 활용:** 분석 결과(권장 사항)를 **Amazon S3로 익스포트**하여 추가적인 분석이나 보고에 활용할 수 있습니다.

---

## 3. 메모리(RAM) 분석을 위한 필수 설정

Compute Optimizer가 RAM 사용량을 분석하고 추천을 제공하기 위해서는 추가 설정이 필요합니다.

### **CloudWatch 에이전트 설치**

* **이유:** 기본적으로 EC2는 CPU, 네트워크, 디스크 I/O 지표만 CloudWatch로 보냅니다. 메모리 데이터는 운영체제 내부 지표이므로 에이전트가 필요합니다.
* **과정:**
  1. EC2 인스턴스에 **CloudWatch 에이전트** 설치 및 실행.
  2. 에이전트가 **RAM 지표**를 CloudWatch로 전송.
  3. Compute Optimizer가 해당 데이터를 가져와 분석 후 권장 사항 제공.



### **지표별 에이전트 필요 여부**

| 분석 대상 | CloudWatch 에이전트 필요 여부 |
| --- | --- |
| **CPU 사용률** | 불필요 (기본 제공) |
| **Network (In/Out)** | 불필요 (기본 제공) |
| **Disk (Read/Write Ops)** | 불필요 (기본 제공) |
| **RAM (메모리 활용도)** | **필수 설치** |

---

> **핵심 요약:**
> Compute Optimizer는 머신러닝으로 CloudWatch 데이터를 분석해 리소스 크기를 제안합니다. 특히 **메모리 기반 추천**을 받고 싶다면 반드시 **CloudWatch Agent**를 심어야 한다는 점을 꼭 기억하세요!
