## 1. 핵심 개념 정의: 내구성 vs 가용성

스토리지 클래스를 선택할 때 가장 먼저 고려해야 할 두 가지 지표입니다.

| 구분 | 정의 | S3 성능 수준 |
| --- | --- | --- |
| **내구성 (Durability)** | 데이터가 손실되지 않고 유지되는 비율 | **99.999999999% (11 9s)**<br>

<br>모든 스토리지 클래스에 동일하게 적용 |
| **가용성 (Availability)** | 필요할 때 즉시 데이터에 액세스할 수 있는 비율 | 클래스별로 상이 (99.5% ~ 99.99%)<br>

<br>낮을수록 비용 저렴 |

> **참고:** 11 9s의 내구성이란 1,000만 개의 객체를 저장했을 때 1만 년에 1개가 손실될 정도로 매우 높은 신뢰도를 의미합니다.

## 2. S3 스토리지 클래스 상세

### ① S3 Standard (General Purpose)

* **특징:** 기본 스토리지 클래스. 지연 시간이 짧고 처리량이 높음.
* **가용성:** 99.99%
* **사용 사례:** 자주 액세스하는 데이터, 빅데이터 분석, 모바일/게임 앱, 콘텐츠 배포.

### ② S3 Standard-IA (Infrequent Access)

* **특징:** 자주 액세스하지 않지만 필요할 때 즉시(밀리초 단위) 액세스해야 하는 데이터용.
* **비용 구조:** 저장 비용은 Standard보다 저렴하지만, **데이터 검색(Retrieval) 비용**이 발생.
* **가용성:** 99.9%
* **사용 사례:** 재해 복구(DR), 장기 백업.

### ③ S3 One Zone-IA

* **특징:** 데이터를 **단일 가용 영역(AZ)**에만 저장. AZ 자체가 파괴되면 데이터 손실 위험 있음.
* **가용성:** 99.5% (가장 낮음)
* **사용 사례:** 재생성 가능한 데이터의 보조 백업, 온프레미스 데이터의 복제본.

## 3. S3 Glacier 클래스 (아카이브용)

장기 보관을 목적으로 하며, 비용이 매우 저렴합니다.

| 클래스 | 특징 | 검색 시간 (Retrieval Time) | 최소 유지 기간 |
| --- | --- | --- | --- |
| **Instant Retrieval** | 분기별 1회 액세스 데이터 | **밀리초(ms) 단위** | 90일 |
| **Flexible Retrieval** | 기존 S3 Glacier | 1~5분(Expedited), 3~5시간(Standard), 5~12시간(Bulk-무료) | 90일 |
| **Deep Archive** | 가장 저렴한 비용 | 12시간(Standard), 48시간(Bulk) | 180일 |

## 4. S3 Intelligent-Tiering

사용자의 개입 없이 **자동으로 비용을 최적화**해주는 클래스입니다.

* **작동 방식:** 액세스 패턴을 모니터링하여 객체를 가장 경제적인 티어로 자동 이동시킵니다.
* **비용:** 별도의 모니터링 및 자동화 수수료가 매달 발생하지만, **검색 비용이 없습니다.**
* **티어 구성:**
1. **Frequent Access (기본):** 자주 사용하는 데이터.
2. **Infrequent Access:** 30일간 액세스 없을 시 이동.
3. **Archive Instant Access:** 90일간 액세스 없을 시 이동.
4. **Archive Access (선택):** 90~700일 이상 액세스 없을 시.
5. **Deep Archive Access (선택):** 180~700일 이상 액세스 없을 시.



---

## 5. 요약 및 관리 방법

* **수명 주기 구성 (Lifecycle Configuration):** 설정된 규칙에 따라 객체를 자동으로 다른 클래스로 전환하거나 삭제할 수 있습니다.
* **수동 변경:** 객체 생성 시 클래스를 선택하거나, 나중에 직접 수정할 수도 있습니다.
