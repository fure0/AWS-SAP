## 🛡️ AWS Firewall Manager 상세 정리

AWS Firewall Manager는 AWS Organizations 내의 **여러 계정과 리소스에 대한 방화벽 규칙을 중앙에서 관리하고 배포**하는 보안 관리 서비스입니다.

### 1. 핵심 개념

* **중앙 집중식 관리:** 수많은 계정의 방화벽 규칙을 일일이 설정할 필요 없이 한 곳에서 제어합니다.
* **보안 정책(Security Policy):** 조직 전체에 공통으로 적용할 보안 규칙의 집합을 정의합니다.
* **자동 적용:** 새로운 리소스(예: 새 ALB)가 생성되면 Firewall Manager가 정책에 따라 자동으로 방화벽 규칙을 할당합니다.

### 2. 관리 가능한 주요 서비스 및 리소스

Firewall Manager는 단독으로 동작하기보다 다음 서비스들의 규칙을 배포하는 '컨트롤 타워' 역할을 합니다.

| 분류 | 관련 서비스 | 적용 리소스 |
| --- | --- | --- |
| **웹 보안** | **AWS WAF** | ALB, API Gateway, CloudFront |
| **DDoS 보호** | **AWS Shield Advanced** | ALB, CLB, NLB, Elastic IP, CloudFront |
| **네트워크 보안** | **보안 그룹(Security Groups)** | EC2, ALB, VPC ENI |
| **인프라 보안** | **AWS Network Firewall** | VPC 수준의 네트워크 트래픽 |
| **DNS 보안** | **Amazon Route 53 Resolver DNS Firewall** | VPC DNS 쿼리 보호 |

### 3. 주요 특징 및 작동 방식

* **리전 수준 생성:** 보안 정책은 리전별로 생성되지만, AWS Organizations를 통해 조직 내의 **모든 계정**에 전파될 수 있습니다.
* **규정 준수 강제:** 특정 리소스가 정책을 위반하거나(예: WAF가 누락된 ALB), 새로운 리소스가 생성될 경우 자동으로 규칙을 적용하여 보안 표준을 유지합니다.

## ⚔️ 서비스 비교: WAF vs Shield vs Firewall Manager

세 서비스는 서로 경쟁하는 것이 아니라 **상호 보완적**인 관계입니다.

### **① AWS WAF (Web Application Firewall)**

* **목적:** SQL 인젝션, XSS 등 웹 애플리케이션 계층(Layer 7) 공격 방어.
* **사용 시점:** 단일 리소스나 특정 계정 내에서 세밀한 웹 ACL 규칙이 필요할 때.

### **② AWS Shield Advanced**

* **목적:** 정교한 DDoS 공격으로부터 인프라 보호.
* **추가 혜택:** * **SRT(Shield Response Team):** DDoS 전문가의 직접 지원.
* **비용 보호:** DDoS 공격으로 인한 비용 급증 방지.
* WAF 규칙 자동 생성 기능 포함.



### **③ AWS Firewall Manager**

* **목적:** WAF와 Shield Advanced 등을 **대규모(Multi-account)**로 배포하고 관리.
* **사용 시점:** 여러 계정에 걸쳐 WAF 규칙을 통일하거나, 모든 계정에 Shield Advanced를 강제로 적용하고 싶을 때.
