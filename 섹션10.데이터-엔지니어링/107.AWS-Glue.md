## ## AWS Glue 개요

AWS Glue는 데이터 분석을 시작하기 전, 데이터를 준비하고 변환하는 데 최적화된 **완전 관리형 서버리스 ETL 서비스**입니다.

* **ETL 의미:**
* **Extract (추출):** 소스에서 데이터를 가져옴
* **Transform (변환):** 데이터를 분석에 적합한 형태로 변경
* **Load (로드):** 대상 시스템에 데이터를 저장



---

## 1. Glue ETL: 데이터의 변환과 이동

Glue ETL은 서버를 직접 관리할 필요 없이 데이터의 흐름을 제어합니다.

* **작동 방식:**
1. **데이터 추출:** Amazon S3, Amazon RDS 등 소스에서 데이터를 가져옵니다.
2. **데이터 변환:** Glue ETL 내에서 비즈니스 로직에 맞게 데이터를 가공합니다.
3. **데이터 로드:** 가공된 데이터를 Amazon Redshift와 같은 데이터 웨어하우스나 다른 저장소로 보냅니다.



---

## 2. Glue Data Catalog & Crawler: 데이터의 지도

데이터가 어디에 어떤 형태로 있는지 파악하는 역할을 합니다.

### **Glue Crawler (크롤러)**

* 다양한 데이터 소스(S3, RDS, DynamoDB, JDBC 호환 DB 등)를 직접 스캔합니다.
* 데이터의 구조(스키마)를 자동으로 감지하여 **메타데이터**를 생성합니다.

### **Glue Data Catalog (데이터 카탈로그)**

* 크롤러가 수집한 메타데이터(테이블 이름, 열 이름, 데이터 타입 등)를 중앙에서 관리하는 저장소입니다.
* Glue ETL 작업 시 이 카탈로그를 참조하여 데이터를 정확히 찾아내고 처리합니다.

---

## 3. 타 서비스와의 연동 (데이터 허브 역할)

Glue Data Catalog는 단순히 Glue만 쓰는 것이 아니라, AWS의 다양한 분석 서비스와 긴밀하게 연결됩니다.

| 서비스명 | Glue Data Catalog 활용 방식 |
| --- | --- |
| **Amazon Athena** | 카탈로그의 메타데이터를 사용하여 S3의 데이터를 SQL로 쿼리함 |
| **Redshift Spectrum** | 데이터 웨어하우스 외부(S3)에 있는 데이터를 조회할 때 참조 |
| **Amazon EMR** | 빅데이터 분석 시 카탈로그를 중앙 메타스토어로 활용 |

---

> **핵심 요약:** > AWS Glue는 **Crawler**를 통해 데이터의 형태를 파악하고, 그 정보를 **Data Catalog**에 저장한 뒤, **ETL 작업**을 통해 데이터를 변환하여 필요한 곳으로 보내주는 분석의 전초기지입니다.
