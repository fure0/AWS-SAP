## 1. Amazon EKS란 무엇인가?

**Amazon EKS**는 AWS에서 제공하는 **관리형 Kubernetes(K8s) 서비스**입니다.

* **Kubernetes(K8s):** 컨테이너화된 애플리케이션의 배포, 스케일링, 관리를 자동화하는 오픈소스 시스템입니다.
* **ECS vs EKS:**
  * **ECS:** AWS 전용 서비스 (독자 API 사용, 비오픈소스).
  * **EKS:** 오픈소스 표준 기반 (Kubernetes API 사용). 다양한 클라우드 제공업체(Azure, GCP 등)에서 공통으로 사용되는 표준화된 방식입니다.



---

## 2. EKS의 주요 특징 및 활용 사례

### 💡 주요 특징

* **Cloud Agnostic (클라우드 불가지론):** 특정 클라우드에 종속되지 않습니다.
* **유연한 마이그레이션:** 온프레미스나 타 클라우드에서 실행 중인 Kubernetes 환경을 AWS로 옮기기가 매우 수월합니다.

### 🏃 활용 사례

* 이미 온프레미스 또는 타 클라우드에서 Kubernetes를 사용 중인 경우.
* Kubernetes 표준 API를 그대로 유지하고 싶은 경우.
* AWS 인프라를 통해 Kubernetes 클러스터 관리 부담을 줄이고 싶은 경우.

---

## 3. EKS의 구성 요소와 실행 모드

EKS의 기본 단위와 실행 방식은 다음과 같습니다.

### 🏗️ 핵심 용어

* **노드(Node):** 컨테이너가 실행되는 EC2 인스턴스 (워커 노드).
* **포드(Pod):** Kubernetes의 가장 작은 배포 단위. (ECS의 '태스크'와 유사한 개념)

### ⚙️ 실행 모드 (Compute)

| 구분 | EC2 실행 모드 | Fargate 실행 모드 |
| --- | --- | --- |
| **특징** | EC2 인스턴스를 워커 노드로 직접 배포 | **서버리스** 컨테이너 배포 방식 |
| **관리** | 사용자가 인스턴스 관리/패치 필요 | AWS가 인스턴스 관리 (노드 관리 불필요) |
| **장점** | 세부적인 커스터마이징 가능 | 운영 오버헤드 최소화 |

---

## 4. 노드 유형 및 관리 방식

EKS에서 워커 노드를 관리하는 세 가지 방법이 있습니다.

1. **관리형 노드 그룹 (Managed Node Groups):**
   * AWS가 사용자를 대신해 EC2 인스턴스(노드)를 생성 및 관리합니다.
   * Auto Scaling 그룹의 일부로 작동하며, 온디맨드 및 스폿 인스턴스를 모두 지원합니다.

2. **자체 관리형 노드 (Self-Managed Nodes):**
   * 사용자가 직접 노드를 생성하고 EKS 클러스터에 등록합니다.
   * 고도의 커스터마이징이 필요한 경우 사용하며, **EKS Optimized AMI**를 사용해 구축 시간을 단축할 수 있습니다.

3. **AWS Fargate:**
   * 노드를 전혀 관리할 필요가 없는 완전 관리형 서버리스 옵션입니다.



---

## 5. 데이터 볼륨 및 스토리지 (Storage)

EKS 클러스터에 데이터를 저장하려면 **StorageClass** 매니페스트와 **CSI(Container Storage Interface)** 호환 드라이버를 사용해야 합니다.

### 📁 지원되는 스토리지 서비스 (시험 키워드)

* **Amazon EBS:** 일반적인 블록 스토리지.
* **Amazon EFS:** **Fargate 모드와 함께 작동하는 유일한 스토리지 클래스**입니다. (중요)
* **Amazon FSx for Lustre / FSx for NetApp ONTAP:** 고성능 병렬 파일 시스템 지원.
