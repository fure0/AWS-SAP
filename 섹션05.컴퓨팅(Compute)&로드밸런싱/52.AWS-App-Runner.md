## 1. AWS App Runner 개요

AWS App Runner는 개발자가 인프라(서버, 네트워크, 컨테이너 오케스트레이션 등)를 관리할 필요 없이 **웹 애플리케이션과 API를 대규모로 빠르게 배포**할 수 있게 해주는 **완전 관리형(Fully Managed)** 서비스입니다.

### 핵심 특징

* **추상화:** 컨테이너 인프라나 복잡한 설정에 대한 지식 없이도 배포 가능합니다.
* **간편한 설정:** 소스 코드나 Docker 이미지를 선택한 후, 기본 사양(vCPU, 메모리, Auto Scaling, Health Check)만 지정하면 됩니다.
* **자동화:** 설정이 완료되면 자동으로 컨테이너 생성, 배포, URL 할당까지 이루어집니다.

### 주요 기능 및 이점

* **자동 확장 (Auto Scaling):** 트래픽에 따라 자동으로 인프라 확장/축소.
* **고가용성 (High Availability):** 결함 내성이 있는 인프라 제공.
* **로드 밸런싱 및 암호화:** 트래픽 분산과 보안 연결 기본 제공.
* **VPC 액세스:** VPC 내부의 리소스(DB, Cache, Message Queue 등)에 안전하게 연결 가능.
* **빠른 프로덕션 배포:** 마이크로서비스 및 API 배포에 최적화.

## 2. App Runner 다중 지역(Multi-Region) 아키텍처

글로벌 서비스 구현을 위해 여러 리전에 App Runner를 배치하고 데이터를 동기화하는 전략입니다.

### 구성 요소 및 흐름

1. **이미지 관리 (Amazon ECR):**
   * Docker 이미지를 ECR 레지스트리에 저장합니다.
   * **ECR 리전 간 복제(Cross-Region Replication)** 기능을 사용하여 이미지를 `us-east-1`에서 `us-west-2` 등 다른 리전으로 자동 복제합니다.

2. **데이터 계층 (Amazon DynamoDB):**
   * **DynamoDB Global Tables**를 사용하여 리전 간 데이터를 실시간으로 복제합니다.
   * 각 리전의 App Runner 서비스는 가장 가까운 리전의 DynamoDB 엔드포인트에 연결됩니다.

3. **트래픽 라우팅 (Amazon Route 53):**
   * **지연 시간 기반 라우팅(Latency-based Routing)** 레코드를 설정합니다.
   * 사용자가 접속하면 Route 53이 각 리전의 App Runner 엔드포인트 중 지연 시간이 가장 짧은(가장 가까운) 곳으로 리디렉션합니다.



### 아키텍처 요약 표

| 구성 요소 | 역할 | 비고 |
| --- | --- | --- |
| **App Runner** | 애플리케이션 실행 | 리전별 독립적 배포 |
| **ECR** | 컨테이너 이미지 저장 | 리전 간 복제 설정 가능 |
| **DynamoDB** | 글로벌 데이터 저장소 | Global Tables로 데이터 동기화 |
| **Route 53** | 트래픽 제어 | 지연 시간 기반 라우팅 적용 |

---

> **Tip:**
> App Runner는 인프라 관리를 최소화하면서도 기업급 보안과 확장성을 챙겨야 할 때 아주 유용한 서비스입니다.
