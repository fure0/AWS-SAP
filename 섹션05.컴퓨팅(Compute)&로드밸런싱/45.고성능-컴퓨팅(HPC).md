# 🚀 AWS 고성능 컴퓨팅 (HPC) 완벽 정리

HPC(High Performance Computing)는 클라우드의 유연함(필요할 때 생성, 사용 후 삭제, 비용 최적화)을 가장 잘 활용할 수 있는 분야입니다. 유전학, 기상 예보, 금융 위험 모델링, 머신러닝 등에 주로 사용됩니다.

---

## 1. 데이터 전송 (Data Management)

HPC는 대규모 데이터를 다루므로 효율적인 전송 수단이 필수적입니다.

| 서비스 | 주요 특징 | 용도 |
| --- | --- | --- |
| **Direct Connect** | 프라이빗 보안 네트워크 (GBps 속도) | 안정적인 전용선 연결 필요 시 |
| **Snowball / Snowmobile** | 물리적 데이터 전송 장치 (PB 단위) | 대규모 데이터를 한 번에 옮길 때 |
| **DataSync** | NFS/SMB → S3, EFS, FSx 전송 서비스 | 온프레미스와 AWS 간 데이터 동기화 |

---

## 2. 컴퓨팅 및 네트워킹 (Compute & Networking)

계산 능력과 인스턴스 간의 통신 속도가 HPC의 핵심입니다.

### 🔹 인스턴스 구성

* **인스턴스 타입:** 작업 특성에 따라 **CPU 최적화(C 시리즈)** 또는 **GPU 최적화(P/G 시리즈)** 선택.
* **비용 최적화:** **스팟 인스턴스(Spot Instances)** 또는 스팟 플릿을 사용하여 비용을 최대 90% 절감.
* **배치 그룹(Placement Groups):** 최적의 네트워크 성능을 위해 **클러스터(Cluster)** 유형 사용 (동일 AZ 내 동일 랙 배치 → 낮은 지연 시간, 10Gbps 이상 속도).

### 🔹 향상된 네트워킹 (Enhanced Networking)

더 높은 대역폭과 낮은 지연 시간(Latency)을 위해 **SR-IOV** 기술을 사용합니다.

1. **ENA (Elastic Network Adapter):** * 최대 **100Gbps** 속도 지원.
* 최신 인스턴스에서 권장되는 표준 방식.


2. **Intel 82599 VF:** * 최대 10Gbps 지원.
* 구형 인스턴스에서 사용되는 레거시 방식.



### 🔹 EFA (Elastic Fabric Adapter) **[시험 중요!]**

* **특징:** HPC 전용으로 개선된 어댑터, **오직 Linux**에서만 작동.
* **핵심 기술:** **OS 우회(OS Bypass)** 기능을 통해 인스턴스 간 통신 시 커널을 거치지 않고 직접 통신.
* **활용:** **MPI(Message Passing Interface)** 표준을 사용하는 밀접하게 연결된(Tightly Coupled) 워크로드에 최적.

---

## 3. 스토리지 (Storage)

HPC의 데이터 처리 속도를 뒷받침하기 위한 고성능 스토리지 옵션입니다.

* **Instance Store:** 인스턴스에 물리적으로 부착된 스토리지. **수백만 IOPS**와 매우 낮은 지연 시간을 제공하지만 휘발성(인스턴스 종료 시 데이터 삭제)임.
* **EBS (io2 Block Express):** 최대 **256,000 IOPS**까지 확장 가능한 고성능 블록 스토리지.
* **Amazon EFS:** 공유 파일 시스템. 프로비저닝된 IOPS 모드를 통해 성능 향상 가능.
* **FSx for Lustre:** **[HPC 전용]** Linux 클러스터용 고성능 파일 시스템. 수백만 IOPS를 제공하며 백엔드에서 S3와 통합됨.

---

## 4. 자동화 및 오케스트레이션 (Orchestration)

* **AWS Batch:** 다중 노드 병렬 작업을 실행하고 관리. EC2 및 스팟 인스턴스 실행 일정을 자동으로 조정하여 효율적인 배치 작업 수행.
* **AWS ParallelCluster:** AWS에서 HPC 클러스터를 신속하게 배포할 수 있는 **오픈 소스 관리 툴**. 텍스트 설정 파일을 통해 VPC, 서브넷, 인스턴스 타입 등 전체 인프라 생성을 자동화함.

---

## 💡 요약: 시험에 자주 나오는 차이점

> **ENA vs EFA?** > 일반적인 고대역폭 네트워킹이 필요하다면 **ENA**,
> MPI를 사용하고 노드 간 지연 시간을 최소화해야 하는 밀접 결합형 HPC라면 **EFA**를 선택하세요!
