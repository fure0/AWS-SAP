## ☁️ AWS Local Zones (로컬 영역) 핵심 정리

### 1. 로컬 영역이란?

AWS 리전을 특정 지리적 위치(도시 등)로 확장하여, **최종 사용자에게 물리적으로 더 가까운 곳**에 컴퓨팅, 스토리지, 데이터베이스 등의 서비스를 배치하는 개념입니다.

* **주 목적:** 레이턴시(지연 시간)에 민감한 애플리케이션 실행.
* **작동 방식:** 기존 AWS 리전을 하나 이상의 '로컬 영역'으로 확장하여 사용.
* **특징:** 사용자 근처에서 실행되어 **한 자릿수 밀리초(millisecond)** 수준의 초저지연 성능을 제공.

### 2. 지원 서비스 및 호환성

로컬 영역은 다음과 같은 주요 AWS 서비스와 호환됩니다:

* **Compute:** EC2, ECS, EKS
* **Storage:** EBS
* **Database/Caching:** RDS, ElastiCache
* **Networking:** Direct Connect

---

### 3. 리전 확장 예시 (us-east-1)

노스 버지니아 리전(`us-east-1`)을 예로 들면, 기본적으로 6개의 가용 영역(AZ)이 있지만 이를 다음과 같은 대도시로 확장할 수 있습니다.

* **추가 가능 지역:** 보스턴, 시카고, 댈러스, 휴스턴, 마이애미 등

> **구조 이해:** > 리전 내에 VPC를 생성한 후, 해당 VPC의 범위를 로컬 영역까지 확장하여 AZ와 로컬 영역을 가로지르는 네트워크를 구성할 수 있습니다.

---

### 4. 콘솔 설정 및 활성화 방법 (Step-by-Step)

#### **① 영역 활성화 (Enable Zones)**

1. EC2 콘솔의 **[Settings]** -> **[Zones]** 메뉴로 이동합니다.
2. 기본 AZ 외에 사용 가능한 **Local Zones** 목록을 확인합니다. (예: 아일랜드 리전은 로컬 영역이 없을 수 있으나, 노스 버지니아는 다수 존재)
3. 원하는 지역(예: Boston)의 로컬 영역을 선택하고 **[Enable]**을 클릭하여 활성화합니다.

#### **② 서브넷 생성 및 연결**

1. VPC 설정에서 새로운 서브넷을 생성합니다.
2. **Availability Zone** 선택 시, 활성화한 로컬 영역(예: `us-east-1-bos-1a`)을 선택합니다.
3. 해당 서브넷에 CIDR 블록을 할당합니다.

#### **③ 인스턴스 배포**

1. EC2 인스턴스 생성 시, 네트워크 설정에서 방금 만든 **로컬 영역 서브넷**을 선택합니다.
2. 이제 해당 EC2 인스턴스는 리전 본체가 아닌, 보스턴에 있는 로컬 영역 인프라 위에서 실행됩니다.

---

### 💡 핵심 요약

* **Local Zones**는 리전의 가용성을 특정 도시 수준으로 확장하는 것입니다.
* 사용자와 물리적 거리를 좁혀 **초저지연(Low Latency)** 환경을 구축할 때 사용합니다.
* VPC 내에 로컬 영역 전용 서브넷을 생성함으로써 간단히 확장할 수 있습니다.
