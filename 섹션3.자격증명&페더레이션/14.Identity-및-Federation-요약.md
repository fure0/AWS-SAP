## 🛡️ AWS 계정 및 거버넌스 관리

다중 계정 환경을 안전하게 구축하고 관리하는 핵심 서비스들입니다.

* **AWS Organizations**: 여러 AWS 계정을 통합 관리하며, 일괄 결제 및 계정 생성을 지원합니다.
* **AWS Control Tower**: 안전하고 규정을 준수하는 **다중 계정 환경(Landing Zone)**을 설정하는 가장 쉬운 방법입니다.
* **가드레일(Guardrails)**: 특정 계정의 비준수 사항을 방지하거나 태깅을 통해 감지하여 보안 표준을 유지합니다.

## 🔑 자격 증명 페더레이션 (Federation)

외부 ID 제공자(IdP)를 통해 AWS에 접속하는 방법들입니다. 시험에서는 **'현대적인 방식'**을 선택하는 것이 포인트입니다.

| 구분 | 서비스/방식 | 주요 특징 |
| --- | --- | --- |
| **최신 권장** | **AWS IAM Identity Center (구 SSO)** | Organizations와 연동하여 **다수 계정 및 SAML 앱**에 통합 접속하는 가장 현대적인 방식입니다. |
| **표준 방식** | **SAML 2.0** | 기업용 IdP와 연동하는 전통적이고 가장 널리 쓰이는 표준 방식입니다. |
| **사용자 지정** | **STS GetFederationToken** | SAML이 없는 경우, 커스텀 브로커를 직접 구축하여 자격 증명을 위임할 때 사용합니다. |
| **B2C/모바일** | **Amazon Cognito** | 앱 사용자 대상. 익명 모드, MFA 등 고급 기능을 제공하며, **기존의 웹 자격 증명 페더레이션보다 권장**됩니다. |

## 📂 AWS Directory Service (Active Directory)

AD 환경을 AWS 내에서 어떻게 구현할지에 대한 선택지입니다.

* **Managed Microsoft AD**: AWS에서 관리하는 실제 MS Active Directory입니다. 온프레미스와의 **신뢰 관계(Trust Relationship)** 설정, MFA, RDS 연동, 심리스 도메인 조인 등을 지원합니다.
* **AD Connector**: 온프레미스 AD에 인증 요청을 전달하는 **디렉터리 프록시**입니다. AWS에 정보를 저장하지 않습니다.
* **Simple AD**: 저렴한 독립형 AD 호환 서비스입니다. 규모가 작고 MFA 같은 고급 기능이 필요 없을 때 적합합니다.

## 📡 리소스 공유

* **AWS Resource Access Manager (RAM)**: 계정 간에 리소스(Subnet, Transit Gateway 등)를 안전하게 공유해야 할 때 사용합니다.

## 💡 시험 합격을 위한 핵심 전략 (Exam Tips)

강의 대본에서 강조된 **'우선순위'**를 기억하세요:

1. **현대화(Modernization)**: 문제에서 옵션이 주어진다면 구식 방식(SAML 직접 연동)보다 **SSO(Identity Center)**가, 단순 웹 페더레이션보다 **Cognito**가 정답일 확률이 높습니다.
2. **용도 구분**: 기업 내부 직원용은 **SSO/AD**, 외부 앱 사용자는 **Cognito**를 선택하세요.
3. **복습 필수**: 이 섹션은 SAP 시험의 근간이 되므로, 확신이 생길 때까지 반복 학습이 필요합니다.
