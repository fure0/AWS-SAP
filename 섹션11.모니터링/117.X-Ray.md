## 🔍 AWS X-Ray 개요

AWS X-Ray는 분산 애플리케이션의 분석 및 디버깅을 위한 서비스입니다. 특히 **마이크로서비스 아키텍처(MSA)**에서 데이터의 흐름을 시각화하고 문제를 추적하는 데 최적화되어 있습니다.

### 주요 기능 및 이점

* **시각적 분석:** 애플리케이션 구성 요소 간의 연결 상태를 서비스 맵으로 제공합니다.
* **요청 추적 (Tracing):** 클라이언트의 요청이 어떤 마이크로서비스를 거쳐 이동하는지 전체 경로를 추적합니다.
* **성능 모니터링:** 각 서비스별 **지연 시간(Latency)** 및 응답 속도를 파악할 수 있습니다.
* **오류 진단:** 어느 지점에서 오류가 발생했는지, 오류 비율은 어느 정도인지 정확히 판단하여 문제 해결 시간을 단축합니다.

---

## 🛠 서비스별 통합 방법

X-Ray를 사용하기 위해서는 각 컴퓨팅 환경에 맞는 설정이 필요합니다.

| 서비스 | 구현 방법 |
| --- | --- |
| **EC2** | **X-Ray 에이전트**를 직접 설치하여 실행 |
| **ECS** | X-Ray 에이전트를 설치하거나, **X-Ray 전용 도커 컨테이너(Sidecar)**를 실행 |
| **Lambda** | 별도 설치 없이 설정만으로 **즉시 사용 가능** |
| **Elastic Beanstalk** | 콘솔에서 **X-Ray 통합 활성화** 시 에이전트 자동 설치 |
| **API Gateway** | 활성화 시 **504(Time-out)** 등의 시간 초과 오류 디버깅에 유용 |

> **⚠️ 중요 (IAM 권한):** X-Ray 에이전트를 실행하거나 데이터를 전송하는 모든 서비스는 Amazon X-Ray와 통신할 수 있는 **올바른 IAM 권한**이 반드시 부여되어 있어야 합니다.

## 🎯 시험 합격을 위한 Key Point

SAP 시험에서 다음 키워드가 등장하면 정답으로 **AWS X-Ray**를 떠올리세요!

1. **분산 추적 (Distributed Tracing):** 여러 서비스에 걸쳐 있는 요청의 경로를 찾아야 할 때.
2. **요청 수준의 문제 해결:** 특정 요청이 왜 느린지, 어디서 끊겼는지 상세 분석이 필요할 때.
3. **마이크로서비스 가시성:** 수많은 마이크로서비스 간의 상관관계를 파악해야 할 때.
