## 📊 1. CloudWatch Metrics (지표)

CloudWatch는 AWS 리소스 및 애플리케이션에 대한 수치 데이터(지표)를 수집합니다.

* **기본 제공 지표:** 대부분의 AWS 서비스가 기본 제공합니다.
* **EC2 지표 특이사항:**
  * **기본 모니터링:** 5분 간격 해상도.
  * **세부 모니터링 (Detailed Monitoring):** 1분 간격 해상도 (유료).
  * **포함 항목:** CPU, Network, Disk (인스턴스 스토어).
  * **⚠️ 미포함 항목:** **RAM(메모리) 사용량**은 기본 지표가 아닙니다.
  * *해결책:* 통합 CloudWatch 에이전트를 설치하여 **Custom Metrics**로 전송해야 합니다.




* **Custom Metrics (맞춤형 지표):**
  * 표준 해상도: 1분.
  * 고해상도(High-Resolution): 최대 **1초** 간격까지 설정 가능.

## 🚨 2. CloudWatch Alarms (알람)

알람은 지표가 임계치를 넘었을 때 특정 작업을 트리거합니다.

### 주요 트리거 작업 (3가지 핵심)

1. **EC2 Action:** 인스턴스 재부팅(Reboot), 중지(Stop), 종료(Terminate), **복구(Recover)**.
   * *복구(Recover):* 하드웨어 실패 시 인스턴스를 새 호스트로 옮기면서 기존 IP(사설/공인)를 유지하는 매우 유용한 기능.


2. **Auto Scaling:** 인스턴스 수 확장 또는 축소 (Scale-in/out).
3. **SNS 알림:** 이메일 전송, SQS 대기열 전송, Lambda 함수 실행 등.

> **💡 EventBridge 연동:** CloudWatch 알람 이벤트를 EventBridge가 가로채어 Kinesis 데이터 전송, Step Functions 워크플로 실행 등 더 복잡한 자동화를 구현할 수 있습니다.

## 🧪 3. CloudWatch Synthetics Canary

사용자 입장에서 웹사이트나 API의 가용성을 모니터링하기 위해 'Canary'라는 스크립트를 주기적으로 실행합니다.

### 주요 특징

* **사용자 시뮬레이션:** NodeJS 또는 Python 스크립트(Headless Chrome 사용)를 통해 실제 고객의 동작을 재현합니다.
* **확인 항목:** 가용성, 지연 시간(Latency), UI 스크린샷, 로드 타임, HTTP 아카이브(HAR) 파일.
* **장점:** 실제 사용자가 장애를 느끼기 전에 미리 문제를 발견하고 알람을 통해 대응(예: Route 53 레코드 변경)할 수 있습니다.

### 주요 블루프린트(설정 템플릿)

* **Heartbeat Monitoring:** URL 로딩 및 스크린샷 저장.
* **API Canary:** REST API의 읽기/쓰기 함수 테스트.
* **Broken Link Checker:** 웹사이트 내 모든 링크의 유효성 검사.
* **Visual Monitoring:** 베이스라인 스크린샷과 현재 화면을 비교.
* **Canary Recorder / GUI Workflow Builder:** 웹사이트 작업(예: 로그인 폼 입력)을 기록하고 검증하는 스크립트 생성.
