## 1. AWS Resource Access Manager (RAM) 개요

AWS RAM은 보유한 AWS 리소스를 **다른 AWS 계정** 또는 **AWS Organizations 내의 조직/OU**와 안전하게 공유할 수 있게 해주는 서비스입니다.

* **주요 목적:** 리소스 중복 생성을 피하고, 운영 효율성을 높이며, 중앙 집중식 관리를 지원합니다.
* **공유 범위:** 특정 AWS 계정, 또는 조직 전체(Organizations)와 공유 가능합니다.

---

## 2. VPC 서브넷 공유 (가장 중요한 활용 사례)

시험에서 가장 자주 등장하는 시나리오입니다. 특정 계정(Owner)의 VPC 서브넷을 다른 계정(Participant)과 공유하는 방식입니다.

### **특징 및 제한 사항**

* **네트워크 공유:** 참가자들은 공유받은 서브넷 내에서 자신의 리소스(EC2, RDS 등)를 생성할 수 있습니다.
* **격리 원칙:** 참가자는 본인의 리소스만 관리할 수 있으며, **타인이나 소유자의 리소스를 열람, 수정, 삭제할 수 없습니다.**
* **제한:**
  * **보안 그룹(Security Group)**은 공유할 수 없습니다 (각자 생성 및 관리).
  * **기본 VPC(Default VPC)**는 공유할 수 없습니다.
  * 모든 계정은 동일한 **AWS Organizations** 소속이어야 합니다.



### **VPC 공유의 장점**

* **네트워크 단순화:** 여러 계정이 하나의 VPC를 사용하므로 **VPC Peering**이나 **Transit Gateway** 설정 필요성이 줄어듭니다.
* **연결성:** 사설(Private) IP를 통해 계정 간 리소스가 직접 통신할 수 있습니다.
* **보안 그룹 참조:** 계정이 달라도 동일 VPC 내에 있다면 보안 그룹 간 참조가 가능하여 보안 정책 관리가 용이합니다.

## 3. 관리형 접두사 목록 (Managed Prefix Lists) 공유

접두사 목록은 하나 이상의 CIDR 블록 세트를 의미하며, 보안 그룹이나 라우트 테이블 관리를 획기적으로 단순화합니다.

* **고객 관리형 접두사 목록:** 사용자가 직접 정의한 IP 대역 세트입니다.
* **공유 이점:** RAM을 통해 공유하면, 소유자가 목록을 업데이트할 때 이를 참조하는 모든 계정의 보안 그룹/라우트 테이블이 **자동으로 업데이트**됩니다.


* **AWS 관리형 접두사 목록:** AWS 서비스(예: S3, CloudFront)를 위해 AWS가 직접 관리하는 목록입니다. (수정/공유 불가, 참조만 가능)

## 4. Route 53 리졸버 규칙 공유

하이브리드 클라우드나 멀티 계정 환경에서 DNS 쿼리를 효율적으로 처리하기 위해 사용합니다.

* **작동 방식:** 주 계정에서 **아웃바운드 리졸버 규칙**(특정 도메인을 온프레미스 DNS로 전달하는 규칙 등)을 생성합니다.
* **공유 효과:** RAM으로 규칙을 공유하면, 다른 계정의 VPC들도 해당 규칙을 연결(Association)하여 동일한 DNS 포워딩 로직을 사용할 수 있습니다.
* **장점:** DNS 리졸버 규칙을 중앙에서 일괄 관리할 수 있습니다.

## 5. RAM으로 공유 가능한 기타 주요 리소스

목록은 계속 추가되고 있으며, 주요 항목은 다음과 같습니다.

| 카테고리 | 공유 가능 리소스 |
| --- | --- |
| **컴퓨팅/네트워크** | Transit Gateway, EC2 전용 호스트, 용량 예약(Capacity Reservations) |
| **데이터/분석** | Aurora DB 클러스터, AWS Glue(카탈로그, 데이터베이스, 테이블) |
| **보안/인증** | ACM 프라이빗 CA, Network Firewall 정책 |
| **기타** | CodeBuild 프로젝트, Outposts, License Manager 설정, Resource Groups |

---

### **💡 시험 핵심 요약**

1. **VPC 공유**는 네트워크 복잡성을 줄이고 IP 관리를 효율화하는 데 최적이다.
2. 공유 서브넷 내에서 **본인 리소스 외에는 제어할 수 없다**는 점을 기억하라.
3. **관리형 접두사 목록** 공유는 대규모 보안 정책 업데이트를 자동화하는 핵심 도구다.
4. **Route 53 리졸버 규칙** 공유는 멀티 계정 DNS 전략의 핵심이다.
