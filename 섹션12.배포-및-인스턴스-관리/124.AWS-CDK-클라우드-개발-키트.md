## 🏗️ AWS CDK (Cloud Development Kit) 이해하기

**AWS CDK**는 익숙한 프로그래밍 언어를 사용하여 클라우드 인프라를 정의할 수 있는 오픈 소스 소프트웨어 개발 프레임워크입니다.

### 1. 주요 특징

* **익숙한 언어 사용:** YAML이나 JSON 같은 설정 파일 대신 JavaScript, TypeScript, Python, Java, .NET 등 선호하는 언어를 그대로 사용할 수 있습니다.
* **Infrastructure as Code (IaC):** 코드로 인프라를 관리하므로 버전 관리, 테스트, 재사용이 용이합니다.
* **CloudFormation 기반:** 작성된 코드는 최종적으로 **CloudFormation 템플릿(JSON/YAML)**으로 컴파일되어 배포됩니다.

---

### 2. CDK 작동 프로세스

단순히 코드를 짜는 것에 그치지 않고, 실제 인프라가 배포되기까지의 과정은 다음과 같습니다.

1. **코드 작성:** Python, TS 등의 언어로 인프라(Lambda, DynamoDB 등)를 정의합니다.
2. **컴파일 (Synthesis):** CDK CLI를 통해 작성한 코드를 **CloudFormation 템플릿**으로 변환합니다.
3. **배포 (Deployment):** 생성된 템플릿이 AWS CloudFormation 서비스에 전달되어 실제 리소스를 생성합니다.

---

### 3. 왜 CDK를 사용하는가? (장점)

* **타입 안정성:** 컴파일 과정에서 오류를 미리 잡아낼 수 있습니다.
* **익숙한 구조:** 클래스, 메서드 등 프로그래밍 패턴을 그대로 사용하여 더 빠르게 작성할 수 있습니다.
* **논리 제어:** 프로그래밍 언어의 강점인 **for 루프, if 조건문** 등을 활용해 복잡한 인프라 로직을 쉽게 구현합니다.
* **코드 재사용:** 자주 사용하는 인프라 패턴을 라이브러리화하여 공유할 수 있습니다.
* **통합 배포:** 인프라 코드와 애플리케이션 런타임(Lambda 함수 등)이 같은 언어를 공유할 수 있어 관리가 효율적입니다.

| 단계 | 구성 요소 (예시) | 설명 |
| --- | --- | --- |
| **Step 1** | **VPC 정의** | 네트워크 환경 구축 |
| **Step 2** | **ECS Cluster** | 컨테이너 실행 환경 설정 |
| **Step 3** | **ALB + Fargate** | 로드 밸런서와 서버리스 컨테이너 서비스 연결 |
