## 1. 하이이브리드 연결 이중화 (Redundancy) 전략

데이터 센터(On-premises)와 AWS 간의 연결 안정성을 보장하기 위해 다양한 이중화 구성을 사용할 수 있습니다.

### ① VPN 기반 액티브-액티브 (Active-Active) 설정

* **구성:** 두 개의 데이터 센터(DC1, DC2)가 각각 독립적인 VPN 터널을 통해 AWS와 연결됩니다.
* **상호 연결:** DC1과 DC2는 내부 네트워크로 서로 연결되어 있어야 합니다.
* **장점:** * DC1의 VPN 연결이 중단되더라도, 트래픽은 내부망을 통해 DC2로 이동한 뒤 DC2의 VPN을 통해 AWS에 접속할 수 있습니다.
* 두 경로를 모두 사용하여 대역폭을 효율적으로 활용합니다.



### ② Direct Connect(DX) 이중화

* **구성:** 두 개의 서로 다른 Direct Connect 로케이션을 통해 AWS에 연결합니다.
* **복구 메커니즘:** 한쪽 DX 로케이션에 장애가 발생하더라도, 내부 네트워킹을 통해 다른 데이터 센터의 DX 경로로 우회하여 AWS 리소스에 계속 접근할 수 있습니다.

### ③ 혼합 구성 (DX + VPN 백업)

* **구성:** 주 연결은 **Direct Connect**로 설정하고, 비용 절감 및 백업을 위해 **Site-to-Site VPN**을 보조 경로로 구성합니다.
* **특징:** DX 장애 시 즉시 VPN으로 장애 조치(Failover)가 이루어지도록 라우팅을 설정합니다.

## 2. Direct Connect Gateway: SiteLink 기능

SiteLink는 AWS 리전을 거치지 않고 **온프레미스 데이터 센터 간에 직접 통신**할 수 있게 해주는 기능입니다.

### ① SiteLink의 작동 원리

* 일반적으로 DX는 온프레미스와 AWS 리전 간 통신을 위해 사용되지만, SiteLink를 활성화하면 **AWS 글로벌 네트워크 백본**을 사용하여 데이터 센터끼리 데이터를 주고받습니다.
* 데이터 경로는 `데이터 센터 A → DX 로케이션 → AWS 글로벌 네트워크 (리전 우회) → DX 로케이션 → 데이터 센터 B` 순으로 흐릅니다.

### ② 주요 특징 및 장점

* **리전 우회:** 데이터가 AWS 리전 내부로 라우팅되지 않으므로 지연 시간이 단축되고 효율적입니다.
* **글로벌 연결:** 전 세계에 분산된 데이터 센터들을 AWS의 고속 사설 네트워크로 연결할 수 있습니다.
* **간편한 구성:** 데이터 센터 간 별도의 전용선을 깔지 않고도 Direct Connect 게이트웨이에서 옵션을 켜는 것만으로 사설 네트워크망(Full-mesh)을 구축할 수 있습니다.
