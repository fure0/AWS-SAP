# CloudHSM 은 뭐야? HSM은 뭐고?

클라우드 환경에서 보안을 다루다 보면 'HSM'이라는 용어를 자주 접하게 되죠. 쉽게 설명해 드릴게요.

---

## 1. HSM이란? (Hardware Security Module)

**HSM(하드웨어 보안 모듈)**은 디지털 키(암호화 키)를 안전하게 생성, 보관, 관리하기 위해 설계된 **물리적인 컴퓨팅 장치**입니다.

보통 암호화 키를 일반 서버의 소프트웨어 파일로 저장하면 해킹 시 유출될 위험이 크지만, HSM은 다음과 같은 특징으로 보안을 강화합니다.

* **물리적 격리:** 암호 연산을 일반 OS가 아닌 전용 하드웨어 내부에서 처리합니다.
* **변조 방지 (Tamper-resistant):** 누군가 장치를 강제로 열려고 하면 내부 데이터를 스스로 파괴하는 물리적 보호 메커니즘이 작동합니다.
* **고성능 암호화:** 복잡한 수학적 연산을 전용 칩에서 처리하여 속도가 빠릅니다.

---

## 2. CloudHSM이란?

**CloudHSM**은 위에서 설명한 HSM 기능을 **클라우드 서비스(예: AWS, Azure 등)** 환경에서 사용할 수 있도록 제공하는 서비스입니다.

과거에는 기업들이 수천만 원짜리 HSM 장비를 직접 사서 데이터 센터에 설치해야 했지만, 이제는 클라우드에서 클릭 몇 번으로 빌려 쓸 수 있게 된 것이죠.

### 주요 특징

* **단독 점유 (Single-tenant):** 클라우드 업체조차 접근할 수 없는, 고객님만을 위한 전용 하드웨어를 할당받습니다.
* **규제 준수:** 금융권이나 공공기관처럼 FIPS 140-2 Level 3 같은 엄격한 보안 인증이 필요한 경우 필수적으로 사용합니다.
* **직접 제어:** 암호화 키의 생성, 삭제, 권한 관리를 사용자가 완전히 통제합니다.

---

## 3. 한눈에 비교하기

| 구분 | 소프트웨어 방식 (KMS 등) | CloudHSM |
| --- | --- | --- |
| **저장소** | 공유된 가상 환경 | **전용 물리 하드웨어** |
| **보안 수준** | 높음 (논리적 격리) | **최상 (물리적 격리)** |
| **관리 주체** | 클라우드 업체와 공동 관리 | **사용자 독점 관리** |
| **비용** | 비교적 저렴함 | 상대적으로 높음 |

---

> **요약하자면:**
> **HSM**은 '금고' 그 자체이고, **CloudHSM**은 '클라우드 업체가 빌려주는 개인용 디지털 금고'라고 이해하시면 됩니다.
