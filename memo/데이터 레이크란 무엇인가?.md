**데이터 레이크(Data Lake)**는 말 그대로 '데이터의 거대한 호수'를 의미합니다. 정제되지 않은 가공 전의 원천 데이터(Raw Data)를 형식에 상관없이 한곳에 모아두는 저장소라고 이해하시면 됩니다.

비유하자면, 생수를 병에 담아 파는 '편의점'보다는 언제든 필요한 만큼 물을 길어다 정수해서 쓸 수 있는 '거대한 천연 호수'에 가깝습니다.

---

### 1. 주요 특징

* **모든 형식 수용:** 텍스트, 이미지, 로그 파일, 동영상, 데이터베이스 테이블 등 어떤 형태의 데이터도 변환 없이 그대로 저장합니다.
* **저렴한 저장 비용:** 주로 클라우드 스토리지(예: AWS S3)를 활용하여 대용량 데이터를 경제적으로 보관할 수 있습니다.
* **읽기 시 스키마 적용(Schema-on-Read):** 데이터를 넣을 때 형식을 맞추는 게 아니라, 나중에 데이터를 꺼내서 분석할 때 비로소 목적에 맞게 구조를 정의합니다.

---

### 2. 데이터 웨어하우스(DW)와의 차이점

데이터 레이크를 더 잘 이해하려면 기존의 **데이터 웨어하우스**와 비교해 보는 것이 좋습니다.

| 구분 | 데이터 웨어하우스 (Data Warehouse) | 데이터 레이크 (Data Lake) |
| --- | --- | --- |
| **데이터 상태** | 정제 및 가공된 데이터 (선별됨) | 가공되지 않은 원천 데이터 (전체) |
| **저장 방식** | 구조화된 표(Table) 형태 | 파일, 오브젝트 등 자유로운 형태 |
| **유연성** | 구조 변경이 어렵고 경직됨 | 매우 유연하며 확장이 쉬움 |
| **주요 사용자** | 비즈니스 분석가, 운영자 | 데이터 사이언티스트, 데이터 엔지니어 |
| **비유** | 잘 정리된 '생수병 창고' | 거대한 '천연 호수' |

---

### 3. 왜 사용하나요?

최근 기업들이 데이터 레이크를 선호하는 이유는 **빅데이터 분석과 AI(인공지능)** 때문입니다.

* 나중에 어떤 데이터가 중요해질지 미리 알 수 없기 때문에 일단 다 모아둡니다.
* 머신러닝 모델을 학습시킬 때는 정제된 데이터뿐만 아니라 가공되지 않은 방대한 양의 로그나 이미지 데이터가 필요하기 때문입니다.

---

### 4. 주의할 점: 데이터 늪(Data Swamp)

데이터 레이크를 관리 없이 방치하면, 어떤 데이터가 어디에 있는지 찾을 수 없게 되어 쓸모없는 쓰레기만 가득한 **'데이터 늪'**이 될 수 있습니다. 따라서 적절한 **메타데이터(데이터에 대한 설명)** 관리와 거버넌스가 필수적입니다.

혹시 **AWS의 OpenSearch**나 **S3**를 활용한 데이터 아키텍처 관점에서 더 궁금한 부분이 있으신가요? 구체적인 활용 사례를 알고 싶으시다면 관련 내용을 더 자세히 설명해 드릴 수 있습니다.
