# 데이터 베이스와 데이터 웨어하우스의 차이?

## 1️⃣ 목적 차이 (가장 중요)

| 구분   | 데이터베이스 (DB)           | 데이터 웨어하우스 (DW)          |
| ---- | --------------------- | ----------------------- |
| 주 목적 | **실시간 서비스 처리 (OLTP)** | **분석 / 통계 / 집계 (OLAP)** |
| 사용 예 | 로그인, 주문, 결제           | 매출 분석, 리포트, 대시보드        |

---

## 2️⃣ 쉽게 비유하면

* **DB** = 지금 가게에서 쓰는 계산대
* **DW** = 본사에서 보는 매출 분석 시스템

---

## 3️⃣ 구조와 특징 차이

### 🔹 데이터베이스 (예: RDS, MySQL)

* 빠른 **읽기/쓰기**
* 짧고 단순한 쿼리
* 정규화된 구조
* 트랜잭션 중요 (ACID)

예시:

```sql
SELECT * FROM users WHERE id = 1;
```

대표 서비스:

* Amazon RDS
* MySQL
* PostgreSQL

---

### 🔹 데이터 웨어하우스 (예: Redshift)

* 대용량 데이터 분석
* 복잡한 JOIN, 집계
* 컬럼 기반 저장
* 쓰기보다 읽기(분석)에 최적화

예시:

```sql
SELECT region, SUM(amount)
FROM sales
GROUP BY region;
```

대표 서비스:

* Amazon Redshift

---

## 4️⃣ 데이터 흐름

보통 구조는 이렇게 돼:

```
서비스 DB → ETL → 데이터 웨어하우스 → BI 도구
```

운영 DB에서 데이터 복사 → 분석용으로 따로 보관.

왜 분리할까?

👉 분석 쿼리가 무거워서
👉 서비스 DB 성능을 망가뜨릴 수 있음

---

## 5️⃣ 시험에서 자주 나오는 포인트 (SAA 기준)

| 상황         | 정답          |
| ---------- | ----------- |
| 실시간 사용자 처리 | RDS         |
| 대규모 분석     | Redshift    |
| 데이터 레이크    | S3 + Athena |

---

> DW는 "데이터 분석용".
